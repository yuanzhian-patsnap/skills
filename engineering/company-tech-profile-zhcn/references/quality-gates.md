# 质量门控（Quality Gates）

## 运行门控

### 范围门控

- [ ] 公司、主题、时间窗口和决策用途在大范围检索前已冻结
- [ ] 母公司/子公司及别名范围已解析，或已明确缩窄

### 证据门控

- [ ] 主要结论在出现于报告前，已通过 `source_index.csv` 和 `claim_ledger.csv` 登记
- [ ] 在证据基础允许的情况下，专利、论文和公开信号已进行三角验证

### 规范化门控

- [ ] 当专利局偏差、专利族/等同问题或覆盖不足影响结论时，公司计数和路线主张已附加范围说明

### 降级门控

- [ ] 工具层级和覆盖局限已明确说明
- [ ] 运行降级时，置信度表述已相应弱化

### 交付物门控

- [ ] `report.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 在声明完成前均已存在

## 草稿门控

将草稿视为可用前：

- [ ] 公司/主题边界明确，别名已解析
- [ ] 至少执行了 1 次专利检索和 1 次论文检索
- [ ] 主要路线或集群已明确说明并附有证据
- [ ] 报告回答了用户的决策问题
- [ ] 主张台账已存在并覆盖主要结论
- [ ] 每项强结论可追溯至至少一个来源编号
- [ ] 已验证事实 / 证据支持的推断 / 开放差距明确区分

## 正式门控

在宣布交付包完成前：

- [ ] `report.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 均已存在
- [ ] 若专利范围较广，已执行代表性抽样或明确说明跳过原因
- [ ] 若使用了抽样，抽样集被视为代表性样本而非穷举结果
- [ ] 已通过专利或论文证据识别核心技术路线
- [ ] 公司营销主张被视为信号而非事实，除非有佐证
- [ ] 工具层级和覆盖局限已在输出中明确说明
- [ ] 不确定性和降级路径已明确说明
- [ ] 弱信号未被夸大为已确认事实
- [ ] 未漂移至无技术实质的通用公司简介
- [ ] 若证据不足，已明确说明并提供后续建议
- [ ] 可选的 `docx` 或 `pdf` 输出若已声明，实际已完成渲染

## 失败规则

若证据过于薄弱而无法得出强结论，应降低措辞的确定性。不得为使报告显得精炼而强行表达确定性。

主张台账是强主张的最低控制点。
