# 来源路由（Source Routing）

使用实际可用的最高置信度通道。

## 默认优先顺序

1. 结构化专利和论文检索
2. 可用时优先使用领域专项学术或文件数据库
3. `Exa` 用于公开网络发现
4. `Tavily`（可用且有效时）
5. `Brave` 搜索作为额外降级选项
6. `web` 搜索作为通用降级选项
7. `Jina` 或其他已知 URL 阅读器，用于发现后读取页面或 PDF

## 按信息需求路由

| 需求 | 首选通道 | 降级选项 |
| --- | --- | --- |
| 公司别名与法律范围 | 官方材料 + 结构化受让人证据 | `Exa` 然后 `web` |
| 专利足迹 | 结构化专利检索 | `Exa` + 官方域名 `web` |
| 论文足迹 | 结构化论文检索或学术数据库 | `Exa` + `web` |
| 产品或发布信号 | 官方材料 + `Exa` | `web`，然后用 `Jina` 读取页面 |
| 上市公司披露 | 官方文件或财务数据库 | 官方域名 `web` |

## 降级规则

若结构化检索不可用：

- 在 `method_decisions.md` 中说明
- 切换至领域专项数据源、`Exa`、`Tavily`、`Brave`、`web` 和 `Jina`
- 在规模、路线广度和新颖性类结论上适当降低置信度

不得将降级运行伪装成与结构化数据支撑的运行具有相同的证据质量。
