# 质量门控

## 运行门控

### 范围门控

- [ ] 路线集合在大范围检索前已冻结
- [ ] 主题、应用场景、对比层级和时间窗口已明确

### 证据门控

- [ ] 主要路线判断在出现于报告前，已经过 `source_index.csv` 和 `claim_ledger.csv` 的流转
- [ ] 各路线证据在跨路线对比前已独立收集

### 归一化门控

- [ ] `comparison_basis.md` 已存在，并定义了路线范围、对比维度和归一化规则
- [ ] 成熟度或 TRL 语言使用了明确的评分标准

### 反向证据门控

- [ ] 主要建议包含削弱条件或更新触发器

### 降级门控

- [ ] 工具层级和覆盖局限已明确说明
- [ ] 运行降级时，置信度措辞已相应弱化

### 交付物门控

- [ ] 完成声明前，`report.md`、`comparison_basis.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 均已存在

## 草稿门控

将草稿视为可用前：

- [ ] 路线集合在大范围检索前已冻结
- [ ] 对比基准已明确并写入 `comparison_basis.md`
- [ ] 对比层级一致（未混用材料/组件/系统级）
- [ ] 对比矩阵可读，每个单元格含证据
- [ ] 声明台账涵盖主要路线排名和权衡
- [ ] 建议与用户场景挂钩
- [ ] 已核实事实 / 有证据支撑的推断 / 开放缺口已明确区分

## 正式门控

宣布交付包完成前：

- [ ] `report.md`、`comparison_basis.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 均已存在
- [ ] 每条路线至少执行 2 次搜索并分析结果
- [ ] 各路线证据在跨路线对比前已独立收集
- [ ] 若使用 TRL 或成熟度标签，明确评分标准已存在
- [ ] 主要建议的反向证据检验已完成
- [ ] 每项主要声明均有可溯源的引用
- [ ] 排名措辞与证据强度相匹配
- [ ] 工具层级和覆盖局限已明确说明
- [ ] 假设和归一化说明清晰可见
- [ ] 若证据不足以支撑有把握的排名，已明确说明
- [ ] 若声称有可选 `docx` 或 `pdf` 输出，已实际完成渲染

## 失败规则

若路线集合定义不足或证据过于薄弱，降低建议的确定性，并说明什么会改变答案。当对比基准不完整时，不得强行给出确定性排名。

声明台账是强声明的最低控制点。
