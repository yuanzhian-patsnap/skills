# 交付物

## 默认交付物

默认交付物为结构化 Markdown 报告：

- `report.md`
- `comparison_basis.md`

## 各模式报告结构

### 概览模式

1. 执行摘要：主题全景与主要路线族群
2. 路线分类与定义
3. 各路线简介卡
4. 前沿信号与新兴方向
5. 关键证据参考

### 对比模式

1. 执行摘要：推荐哪条路线及原因
2. 范围与对比基准（明确）
3. 各路线简介卡
4. 对比矩阵（路线 × 维度，每个单元格含证据）
5. 建议及条件、权衡与更新触发器
6. 反向证据与削弱条件
7. 关键证据参考

### 成熟度模式

1. 执行摘要：成熟度全景与就绪度差距
2. 范围与成熟度评分标准（明确的级别定义）
3. 各路线成熟度评估（含证据）
4. 成熟度对比矩阵
5. 阶段门或就绪度路线图
6. 风险与部署阻碍因素
7. 关键证据参考

### 机会模式

1. 执行摘要：机会所在
2. 路线全景与当前覆盖情况
3. 差距分析（稀疏覆盖 + 技术价值 + 切入路径）
4. 新兴路线与早期信号
5. 推荐调研优先级
6. 关键证据参考

## 对比矩阵格式

路线（行）× 维度（列）。每个单元格应包含：
- 评估级别（如：高 / 中 / 低，或 TRL 级别）
- 简短证据说明及来源引用

## 可选正式导出

若宿主环境支持渲染，可选导出格式为：

- `deliverables/report.docx`
- `deliverables/report.pdf`

建议渲染路径：

- `pandoc`：从 Markdown 转换为 `docx` 和 `pdf`
- `python-docx`：用于自定义 Word 格式
- `reportlab` 或其他 Markdown 转 PDF 工具：用于生成 `pdf`

这些渲染器为可选项。当 Markdown 和证据文件质量过硬时，本技能即视为完成。
