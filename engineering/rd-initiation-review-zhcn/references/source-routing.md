# 来源路由

使用实际可用的最高置信度通道。

## 默认优先顺序

1. 提案材料和用户提供的文件
2. 结构化专利和论文检索，用于现有技术和前沿证据
3. 可用时，使用领域专项学术或申报数据库
4. `Exa`，用于公开网络发现
5. `Tavily`，在可用且有效时使用
6. `Brave`，作为额外降级方案
7. `web` 检索，作为通用降级方案
8. `Jina` 或其他已知 URL 阅读器，用于发现后读取页面或 PDF

## 按信息需求路由

| 需求 | 首选通道 | 降级方案 |
| --- | --- | --- |
| 提案主张提取 | 提案材料 | 要求补充材料或标注为开放缺口 |
| 新颖性与重叠 | 结构化检索或学术数据库 | `Exa` + `web` |
| 可行性与前沿支撑 | 结构化检索或学术数据库 | `Exa` + `web` |
| 申办方或对比方披露 | 官方申报文件或财务数据库 | 官方域名 `web` |
| 政策、标准及公开部署状态 | `Exa` | `web`，再用 `Jina` |

## 降级规则

若提案材料薄弱或结构化检索不可用：

- 在 `method_decisions.md` 中记录降级情况
- 明确说明哪些内容无法完整核查
- 降低新颖性、重叠和完整性主张的置信度

不得暗示仅凭公开检索即可证明内部重复项目排查的完整性。
