# 质量关口

## 运行关口

### 范围关口

- [ ] 在大范围检索前，已冻结具体项目对象、评审关口和模式
- [ ] 在任何外部检索前，已提取提案材料

### 证据关口

- [ ] 主要评审结论在出现于报告前，已经过 `source_index.csv` 和 `claim_ledger.csv` 的记录
- [ ] 提案陈述、外部佐证、推断和开放缺口四层保持清晰区分

### 归一化关口

- [ ] 创新点在新颖性评估前已归一化为稳定对比集
- [ ] 专业能力/资源和材料完整性作为明确的评审维度处理

### 反证关口

- [ ] 主要建议包含弱化条件、阻碍因素或后续证据需求

### 降级关口

- [ ] 工具层级和覆盖局限性已明确说明
- [ ] 仅凭公开检索的运行不暗示内部重复项目排查的完整性

### 交付物关口

- [ ] 在声明完成前，`report.md`、`novelty-note.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 均已存在

## 草稿关口

### 所有模式

- [ ] 已识别具体项目对象（不仅是主题）
- [ ] 在外部检索前已提取提案陈述主张
- [ ] 维持四层证据分离（提案陈述 / 外部佐证 / 有据推断 / 开放缺口）
- [ ] 已明确说明工具层级和覆盖局限性
- [ ] 已进行材料完整性评估
- [ ] 若内部档案不可用，标注为 `内部查重边界未核验`
- [ ] 当重叠是决策问题时，`novelty-note.md` 已存在
- [ ] 提案主张与外部证据未被随意混淆
- [ ] 主张台账涵盖新颖性、可行性和证据缺口结论

### 预筛查模式附加项

- [ ] 附条件的通过/否决建议已呈现
- [ ] 新颖性边界摘要已呈现
- [ ] 已识别关键缺口和下一步行动

### 标准评审模式附加项

- [ ] 每个创新点至少执行 2 次专利检索和 1 次论文检索
- [ ] 新颖性对比表已呈现
- [ ] 每条主要主张均有可追溯的来源引用

### 创新点评审模式附加项

- [ ] 每个创新点已单独评估
- [ ] 差异化边界表已呈现
- [ ] 每点的残余差异点已明确说明

## 正式关口

### 所有模式

- [ ] `report.md`、`novelty-note.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 均已存在
- [ ] 提案主张与外部证据未被随意混淆

### 专家评审模式附加项

- [ ] 评分标准明确且可追溯
- [ ] 阶段关口矩阵含条件、责任人和后果
- [ ] 反证审查已完成
- [ ] 技术分解涵盖主链、依赖关系和失效模式
- [ ] 若声明可选 `docx` 或 `pdf` 输出，已实际完成渲染

## 失败规则

若证据过于薄弱，无法给出明确的通过/否决声明，
则降低措辞并保持建议为条件性。不得强行给出确定性结论。

主张台账是强结论的最低控制点。
