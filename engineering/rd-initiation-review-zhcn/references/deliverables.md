# 交付物

## 默认交付物

默认交付物为 Markdown 评审包：

- `report.md`
- `novelty-note.md`

## 各模式报告结构

### 预筛查模式（6-8 页当量）

1. 执行建议（推进 / 附条件推进 / 暂缓）
2. 新颖性边界摘要
3. 关键风险与阻碍因素
4. 缺失材料与下一步行动

### 标准评审模式（10-14 页当量）

1. 执行建议
2. 项目范围与提案陈述主张
3. 新颖性与重叠判断（附对比表）
4. 可行性与证据充分性
5. 主要风险与材料缺口
6. 建议与条件

### 创新点评审模式（8-12 页当量）

1. 创新评估执行摘要
2. 逐点创新分析
3. 差异化边界表
4. 创新主张的风险
5. 建议

### 专家评审模式（12-18 页当量）

1. 附置信度的执行建议
2. 项目范围与提案连贯性评估
3. 新颖性与重叠（附详细对比附件）
4. 可行性、证据充分性与技术分解
5. 评分与评级逻辑（附明确评分标准）
6. 阶段关口矩阵
7. 材料缺口与数据就绪度
8. 反证与弱化条件

## 新颖性说明

`novelty-note.md` 应为独立附件，包含：

- 新颖性检索范围（数据库、时间窗口、领域范围）
- 主要对比对象
- 逐点重叠评估
- 残余差异点
- 检索局限性及未检索区域

中文正式报告可命名为 `公开文献与专利预查新说明` 或 `查新与对标附件`。

## 可选正式导出

若宿主环境支持渲染，可选导出格式为：

- `deliverables/report.docx`
- `deliverables/report.pdf`

建议渲染路径：

- `pandoc`：从 Markdown 转换为 `docx` 和 `pdf`
- `python-docx`：自定义 Word 格式
- `reportlab` 或其他 Markdown 转 PDF 工具：生成 `pdf`

这些渲染器为可选项。当 Markdown 和证据文件质量过硬时，本技能即视为完成。
