# 来源路由（Source Routing）

使用实际可用的最高置信度通道。

## 默认优先顺序

1. 结构化专利和论文检索，用于领域发现和受让人证据
2. 可用时，使用领域专属学术或申报数据库
3. `Exa`，用于公开网络发现和官方页面
4. `Tavily`，可用且有效时使用
5. `Brave`，作为额外降级选项
6. `web` 搜索，作为通用降级手段
7. `Jina` 或其他已知 URL 阅读器，用于发现后读取页面或 PDF

## 按信息需求路由

| 需求 | 首选通道 | 降级通道 |
| --- | --- | --- |
| 候选玩家发现 | 结构化检索 + `Exa` | `web` |
| 逐玩家技术证据 | 结构化检索 | `Exa` + 官方域名 `web` |
| 论文与机构归属 | 结构化检索或学术数据库 | `Exa` + `web` |
| 上市公司披露 | 官方申报或财务数据库 | 官方域名 `web` |
| 官方页面、发布及公开信号 | `Exa` | `web`，再用 `Jina` |

## 降级规则

若结构化检索不可用：

- 在 `method_decisions.md` 中说明
- 使用领域专属来源、`Exa`、`Tavily`、`Brave`、`web` 和 `Jina`
- 降低对完整性、玩家排名和路线广度主张的置信度

除非证据真正支持该置信度水平，否则不得将降级运行呈现为完整市场图谱。
