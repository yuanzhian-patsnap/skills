# 质量门控（Quality Gates）

## 运行门控

### 范围门控

- [ ] 领域、地理范围、时间窗口和决策用途在广泛检索前已冻结
- [ ] 候选发现在分层前完成

### 证据门控

- [ ] 行业层面和玩家层面证据保持独立
- [ ] 主要结论在出现于报告前，已经过 `source_index.csv` 和 `claim_ledger.csv` 的流转

### 规范化门控

- [ ] 已发现玩家、已入围玩家和已深度分析玩家未被折叠为一个无差别集合

### 降级门控

- [ ] 工具层级和覆盖局限已明确说明
- [ ] 运行降级时，完整性措辞已相应弱化

### 交付物门控

- [ ] 完成声明前，`report.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 均已存在

## 草稿门控

将草稿视为可用前：

- [ ] 领域范围和地理范围已明确
- [ ] 至少执行了 3 次全行业搜索并分析了结果
- [ ] 候选池在分层前从证据中构建
- [ ] 最终玩家集合可辩护，纳入/排除逻辑已记录
- [ ] 路线或地理主张有来源支撑
- [ ] 主张台账涵盖主要排名和结论
- [ ] 每项强结论可追溯至至少一个来源编号
- [ ] 已验证事实 / 证据支持的推断 / 开放缺口明确区分

## 正式门控

将交付包声明为完成前：

- [ ] `report.md`、`query_log.csv`、`source_index.csv` 和 `claim_ledger.csv` 均已存在
- [ ] 若行业或玩家专利范围较广，已执行代表性采样或明确说明跳过原因
- [ ] 若使用了采样，行业样本支持路线发现而非直接决定最终玩家分层
- [ ] 若使用了玩家采样，每个玩家独立采样
- [ ] 头部玩家逐一深度分析（未跨玩家合并）
- [ ] 技术路线差异化矩阵存在，每个单元格有证据支撑
- [ ] 空白机会主张满足全部 3 个标准（稀疏 + 有价值 + 进入路径），或标注为观察
- [ ] 报告区分已发现玩家与已优先排序玩家
- [ ] 未在无专利、论文或产品证据基础上分配层级
- [ ] 工具层级和覆盖局限在输出中明确说明
- [ ] 弱信号未被夸大为已确认事实
- [ ] 若声称可选 `docx` 或 `pdf` 输出，已实际完成渲染

## 失败规则

若玩家发现不完整或样本过薄，降低措辞力度。证据不支持时，不得强行声称完整性。

主张台账是强主张的最低控制点。
