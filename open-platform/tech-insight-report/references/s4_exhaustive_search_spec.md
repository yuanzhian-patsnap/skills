# §4 专利全景与高风险排查 · 全量检索规范

> 本规范于2026-05-30写入，解决"仅 topk 样本无法支撑高风险结论"的根本缺陷。

## 为什么必须全量检索

| 问题 | 说明 |
|------|------|
| topk 样本遗漏高风险专利 | PatSnap 排序基于相关性，排名靠后的专利可能权利要求最宽 |
| 白点结论不可信 | "0件命中"必须来自全量检索，否则只能说"topk样本内未见" |
| IPC 分布失真 | 样本分布不等于实际申请量分布，趋势图结论存疑 |
| §0 行动矩阵失效 | 高风险排查不完整时，P1/P2 行动建议缺乏依据 |

## 全量检索操作规范（五步法）

```
Step 1：IPC优先策略
  - 列出所有相关IPC分类（B60S1/56 / B08B3/02 / G01S13/00 等）
  - 每个IPC分类单独检索，不合并，记录各自 matched_total
  - matched_total ≤ 500 → 直接全量下载，无需分页
  - matched_total > 500 → 分段：按年份（5年/段）或按子IPC拆分，逐段全量

Step 2：关键词补充策略
  - 每个核心技术词至少设计3种语言变体（中文/英文/缩写）
  - 每个技术方向至少2种关键词组合
  - 两轮检索结果取并集，去重后再排查

Step 3：申请人锁定策略
  - 发现头部申请人后，对其全量专利做申请人检索
  - 确保不遗漏同一申请人名义略有差异的专利

Step 4：近3年新增专项补查
  - date_from=今年-3，date_to=今日，单独跑一遍全量
  - 近期高风险专利（刚授权/刚公开）最易被相关性排序淹没

Step 5：零命中验证协议
  - 若某技术方向 matched_total=0，必须换3组不同关键词二次验证
  - 三次均为0才可在报告中标注"技术白点（检索日期：YYYY-MM-DD，检索策略：XXX）"
  - 禁止将 topk 样本为空直接等同于白点结论
```

## §4章节内容最低标准

```
必须包含：
✅ 全量检索覆盖说明（列出IPC范围、关键词组合、matched_total数字）
✅ IPC分布图（基于全量数据，不得标注"样本"）
✅ 年度趋势图（基于全量申请数，不得用topk近似）
✅ 高风险专利清单（全量中按五步筛选法筛出）
✅ 白点结论（若有）需附三次零命中验证记录

禁止：
❌ 仅用 topk 返回结果绘制趋势图
❌ 仅用 topk 返回结果统计申请人排名
❌ 以"检索样本中未见"代替"全量检索确认为白点"
```

## 与报告其他章节的联动要求

- §4 全量 matched_total 数字必须显示在报告§4开头的"检索概况"框内
- §6 白点结论必须引用§4的三次零命中验证结果，不得独立声明
- §0 行动矩阵的风险等级必须来自§4全量排查后的最终清单
