# 序列检索规范（单抗 FTO 专用）

## 概述

本文档规定在单抗 FTO 分析中使用 `ls_sequence_alignment` MCP 工具执行序列相似度检索的完整操作规范。

## 工具：ls_sequence_alignment

`ls_sequence_alignment` 是 `mcp_tool-collection-biology-modality` 服务器提供的序列比对工具，可在智慧芽专利数据库中检索含有相似生物序列的专利。

## 序列准备

### 输入格式

- 推荐 FASTA 格式，单字母氨基酸序列亦可接受
- 每次调用提交一条序列（VH 和 VL 分开提交）
- 序列必须去除信号肽，使用成熟蛋白序列

### CDR 定义方案

优先使用 **IMGT 编号**（国际通用），备选 Kabat 编号。两种方案均须在报告中明确注明。

**IMGT CDR 位置（重链可变区）：**
- CDR-H1：第 27-38 位
- CDR-H2：第 56-65 位
- CDR-H3：第 105-117 位（可变）

**IMGT CDR 位置（轻链可变区）：**
- CDR-L1：第 27-38 位
- CDR-L2：第 56-65 位
- CDR-L3：第 105-117 位

### 序列提取清单

在开始序列检索前，准备并确认以下序列：

| 序列 ID | 类型 | 来源 | 状态 |
|---|---|---|---|
| VH_001 | 重链可变区全序列 | 候选分子 | 必须 |
| VL_001 | 轻链可变区全序列 | 候选分子 | 必须 |
| CDRH3_001 | CDR-H3 | 从 VH 提取 | 强烈建议 |
| CDRL3_001 | CDR-L3 | 从 VL 提取 | 建议 |
| CDRH1_001 | CDR-H1 | 从 VH 提取 | 可选 |
| CDRH2_001 | CDR-H2 | 从 VH 提取 | 可选 |

## 检索阈值设置

| 序列类型 | 默认阈值 | 适用场景 |
|---|---|---|
| VH 全序列 | ≥ 90% | 标准 FTO 筛查 |
| VL 全序列 | ≥ 90% | 标准 FTO 筛查 |
| CDR-H3 | ≥ 95% 或精确匹配 | 特异性决定区高灵敏检索 |
| CDR-L3 | ≥ 90% | 轻链特异性检索 |
| VH + VL 组合 | 两者均 ≥ 85% | 宽松扩展检索 |

**高风险场景降低阈值（至 80%）的条件：**
- 与已知竞争对手专利高度相似（已知存在风险）
- CDR-H3 仅有 1-2 个残基不同
- 生物类似药审评期间

## 比对结果解读

### 相似度分级

| 相似度范围 | 风险等级 | 处理方式 |
|---|---|---|
| 100% | 极高 | 立即进入深度分析 |
| 95-99% | 高 | 进入深度分析 |
| 90-94% | 中高 | 进入精选列表，评估等同原则 |
| 80-89% | 中 | 保留在专利池，标注观察 |
| < 80% | 低 | 通常不进入精选列表 |

### CDR-H3 特殊规则

CDR-H3 是抗体抗原结合特异性的主要决定区：
- CDR-H3 **精确匹配** = 极高侵权风险，无论其他区域相似度如何
- CDR-H3 **1 个残基差异** = 需要评估等同原则
- CDR-H3 **2+ 个残基差异** = 通常无字面侵权，评估等同原则范围

## 序列侵权分析框架

### 字面侵权判断

当候选分子序列与权利要求限定序列**精确匹配**，或在权利要求描述的序列变体范围内时，构成字面侵权风险。

分析步骤：
1. 提取专利权利要求中的序列限定（SEQ ID NO: X 或 CDR 序列列表）
2. 用 `ls_sequence_alignment` 将候选序列与权利要求序列逐一比对
3. 精确匹配 → 字面侵权风险
4. 部分匹配 → 进入等同原则分析

### 等同原则分析

判断标准（功能-方式-结果三要素，Function-Way-Result）：
- **功能**：候选序列与专利序列是否实现相同功能（如结合相同表位）？
- **方式**：结合方式是否实质相同（如类似的 CDR 接触残基）？
- **结果**：是否获得实质相同的结果（如相似的亲和力和选择性）？

三要素均满足 → 等同原则侵权风险（标注为中-高风险）

### 设计绕开策略（序列层面）

以下修改可能降低序列侵权风险（须结合具体权利要求评估）：

1. **CDR 残基替换**：在不影响靶点结合的关键位置引入保守或非保守替换
2. **框架区改造**：改变非 CDR 区域的序列，同时保留 CDR 构型
3. **CDR 移植**：将 CDR 移植到不同框架区，改变整体序列同一性
4. **表位偏移**：靶向相同靶点的不同表位，规避特定 CDR 序列权利要求
5. **亚型切换**：从 IgG1 改为 IgG4 或其他亚型（规避 Fc-specific 权利要求）

**注意**：所有设计绕开策略须经实验验证仍保留足够活性，并需法律顾问最终确认不侵权。

## 常见数据质量问题

| 问题 | 表现 | 处理方式 |
|---|---|---|
| 序列含信号肽 | 检索结果偏少 | 去除 N 端信号肽后重检 |
| 序列编号错误 | CDR 提取位置偏移 | 用 IMGT/AbNum 重新编号 |
| 混合轻链类型 | κ/λ 混淆 | 分开检索 κ 和 λ 轻链 |
| 双特异性抗体 | 两个 VH/VL 对 | 分别对每个臂检索 |
| ADC 抗体部分 | 同裸单抗处理 | 先完成序列检索再评估 linker/payload 专利 |
