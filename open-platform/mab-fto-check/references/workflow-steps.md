# mab-fto-check 工作流步骤细则

## 步骤总览

```
步骤0  技术解析          智慧芽新药情报库获取序列/靶点/合作伙伴
步骤1  序列检索（M1）    CDR精确→VH/VL 80%+→H/L 90%→ADC额外检Linker/Payload
步骤1.5 修饰检索（M1.5）化学修饰/糖基化/PEG/Fc突变/ADC偶联（ls_structure_search）
步骤2  关键词检索（M2–M8）靶点/表位/平台技术/制剂/联合用药/用途/表达载体
步骤3  竞品企业全量检索  含关联实体挖掘（母子公司/被收购实体/许可方）
步骤4  地域+法律状态过滤+同族合并  PCT空窗期/各国期限补偿/同族拓展/B文本获取
步骤5  两阶段筛选        初筛反向排除 → 精筛权利要求比对
步骤6  侵权判定          全面覆盖原则+等同原则，生成claim_diff_matrix
步骤7  撰写FTO报告       HTML单文件：概述→技术解析→检索策略→CC比对→结论建议→附录A/B/C
```

---

## 步骤0：技术解析

### 0-A 分子信息收集清单
触发条件：用户已填写三段式问卷（见 user-intake-protocol.md）

- [ ] VH 序列（含CDR-H1/H2/H3标注）
- [ ] VL 序列（含CDR-L1/L2/L3标注）
- [ ] H链全长（若有）
- [ ] L链全长（若有）
- [ ] 靶点名称/别名/基因符号
- [ ] 适应症（ICD编码优先）
- [ ] 给药方式（IV/SC）及剂量
- [ ] 抗体类型（IgG1/IgG4/双抗/ADC）
- [ ] 平台技术（人源化/全人/噬菌体/转基因小鼠）
- [ ] 表达系统（CHO/HEK293/SP2/0）
- [ ] 制剂类型（冻干/液态/预充注射器）
- [ ] 联用方案（若有）
- [ ] 竞品药物/公司列表

### 0-E 智慧芽新药情报库查询优先级

```
1. 候选靶点 → 获取已上市同靶点mAb清单
2. 候选分子名/内部编号 → 获取公开序列（若有）
3. 竞品公司 → 获取关联实体（子公司/合作方/被收购实体）
4. 合作伙伴 → 确认现有许可协议是否覆盖目标市场
```

## 步骤1：序列检索细则

### 执行顺序与阈值
```
第1轮（精确）  ：CDR-H3 ≥ 95%
第2轮（高相似）：CDR-H1/H2/L1/L2/L3 ≥ 80%
第3轮（区域）  ：VH全序列 ≥ 85%，VL全序列 ≥ 85%
第4轮（全链）  ：H链全长 ≥ 90%，L链全长 ≥ 90%
第5轮（结构域）：铰链区 ≥ 95%，Fc恒定区 ≥ 95%
```

### ADC额外检索
```
Linker肽段序列 ≥ 90%（Bio数据库）
Payload结构 → 路由chemical数据库（ls_structure_search）
```

## 步骤3：关联实体挖掘规则

```
一级实体：用户指定的竞品公司（直接检索）
二级实体：母公司 / 控股集团（ALL_AN扩展）
三级实体：被收购子公司（近10年并购记录）

```

## 步骤4：同族合并 + 地域/法律状态过滤细则

### 4-A 同族合并（⚠️ 必须在地域过滤后执行）

```
1. 对 patent_pool_filtered.json 中所有专利调用 patsnap_fetch(module=["family"])
2. 按 INPADOC 同族ID（priority_number）分组
3. 每组内按「风险等级降低→申请时间递增」排序，编号 X-1, X-2...
4. 输出 patent_pool_family.json（见 patent-family-merge.md + output-contract.md）
```

### 4-B B文本获取规则

```
对同族内法律状态=granted 的成员：
  → 调用 patsnap_fetch(pn=<B号>, module=["basic"]) 获取完整权利要求
  → 提取独立权利要求存入 representative_claim_text
  → HTML报告中展示 [B文本 ✅] 徽章

对仅有A文本（审中）的同族：
  → 取最新修改版权利要求
  → HTML报告中展示 [A文本/审中 ❌] 徽章
  → 侵权分析段落注明「审查结果不确定性」
```

### PCT空窗期处理
```
对WO申请：标记「PCT空窗期待跟踪」，记录预计进入国家阶段时间，
          报告中列出PCT空窗期专利清单，建议3个月后复查
```

### 各国期限补偿
| 国家 | 补偿机制 | 最长期限 |
|------|----------|----------|
| US | PTA / PTE | PTA无上限；PTE最长5年 |
| JP | 存续期间延长 | 最长5年 |
| CN | 专利权期限补偿 | 最长5年 |
| EP | SPC | 最长5年+6个月儿科 |

## 步骤5：两阶段筛选细则

### 初筛反向排除清单
```
E1: 权利要求范围与候选分子无技术重叠
E2: 靶点/抗原明确不同
E3: 目标市场无同族保护
E4: expiry_date < 今日（已到期，⚠️不纳入失效专利）
E5: 申请人/权利人为己方或合作伙伴
E6: 已知许可协议覆盖（需确认地域和适应症范围）
```

### 精筛权利要求比对标准
```
1. 优先使用B文本独立权利要求（已授权）
2. 拆解为原子技术特征列表
3. 逐特征比对：字面覆盖 → 等同原则
4. 输出 blocking_candidates.json
```

## 步骤6：claim_diff_matrix 生成规范

```
专利号     公开号（含国家前缀，B文本优先）
特征编号   1.1, 1.2, 1.3...
特征描述   权利要求原文摘录（中英双语）
候选对应   简述候选分子的对应技术特征
字面覆盖   ✅/❌/⚠️争议
等同覆盖   ✅/❌/⚠️争议/N/A
结论       覆盖/不覆盖/争议
```

**最终侵权结论规则**：
- 全部特征均「覆盖」→ 侵权风险成立
- 任一特征「不覆盖」且等同不适用 → 不侵权
- 存在「争议」特征 → 中风险，建议法律意见
