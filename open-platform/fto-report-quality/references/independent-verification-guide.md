## 1. 验证目的与展示强制要求

独立验证核心能力：查漏 · 补缺 · 核准 · 增信

> ⚠️ **报告展示强制要求（v3.0新增）**：
> 1. 独立检索验证章节必须作为最终报告的**独立重点章节**，紧跟执行摘要之后、所有评分章节**之前**展示
> 2. 章节标题使用 ⭐【关键章节·请重点核查】标识，与其他章节明显区分
> 3. 漏检专利清单使用红色/橙色警示色标注；有真实数据源链接时附链接，不得虚构链接
> 4. 召回率仪表盘（动态环形图）在网页报告中单独呈现
> 5. Word文档中该章节使用黄色/橙色底纹高亮

## 2. 双轨检索执行规程

### 关键词轨道
- 提取3-8个核心技术术语 + IPC/CPC分类号
- 加入竞争对手申请人（如识别为轨道交通，可考虑西门子/阿尔斯通/川崎/庞巴迪/日立等）

### 语义轨道
- 第一轮：实现层描述（How it works）
- 第二轮（需要时）：效果层描述（What problem it solves）

### 召回率估算（统一采用 Chapman 双样本估算）
```
设：
  A = 被审报告专利池（大小 = n_A）
  B = 独立检索专利池（大小 = n_B）
  C = A ∩ B（重叠数 = n_C）

Chapman估算总量：N_hat = (n_A + 1)(n_B + 1) / (n_C + 1) - 1
报告召回率：R_A = n_A / N_hat
```

若未获得真实独立专利池 B，或无法完成 A/B/C 去重比对，不得计算召回率；报告中填写"未验证"。

| 召回率 | 评级 | A维度影响 | 展示颜色 |
|--------|-----|---------|---------|
| ≥85% | 充分 ✅ | 无扣分 | 🟢绿色 |
| 70-84% | 偏低 ⚠️ | 扣1-2分 | 🟡黄色 |
| <70% | 严重不足 ❌ | 扣3-5分 | 🔴红色 |

### 遗漏专利筛查标准
仅将"高相关 + 法律状态有效 + 初步高/中风险"的专利列入漏检清单

## 4. 高风险专利有效性验证规程

### 4.1 novelty-check（新颖性验证）
触发条件：≥1件高风险专利
可行性判断：高（单一文献全覆盖）/ 中（部分覆盖+公知常识）/ 低（无有力文献）

### 4.2 non-obviousness-check（创造性验证）
触发条件：新颖性可行性为"中"或"高"
评估：最接近现有技术 + 区别特征 + 辅助文献 + 结合动机

### 4.3 有效性稳健度
- **稳固**：新颖性和创造性均无有力攻击
- **存疑**：存在中等强度攻击，建议进一步研究
- **脆弱**：存在高可行性攻击，建议启动无效程序
