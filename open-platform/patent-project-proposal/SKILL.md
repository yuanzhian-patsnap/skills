# 专利分析支撑立项报告生成 Skill

## 触发条件

用户明确提及**项目立项**（如"帮我做立项""支撑立项报告""科研项目立项"等），则启动本 Skill。

---

## 核心工作流

```plain
Step 1: 技术领域界定与检索策略制定
    ↓
Step 2: 执行专利检索（调用智慧芽 MCP）
    ↓
模块1: 国内外研究现状与竞争格局
模块2: 企业现有技术基础与定位
模块3: 创新点与先进性论证
模块4: 知识产权布局规划
模块5: 技术路线选择验证
模块6: 知识产权风险分析（FTO）
模块7: 考核指标设定
    ↓
Step 3: 整合输出完整《科研项目立项报告（专利分析支撑版）》
```

> **重要原则**：每个模块的分析结论必须基于真实专利数据支撑，禁止空泛描述，禁止出现任何费用相关内容。

## Step 1：技术领域界定与检索策略

### 1.1 信息收集（如用户未提供，逐项询问）

| 收集项 | 说明 |
|--------|------|
| 技术领域/关键词 | 核心技术名词（中英文均需） |
| 核心技术问题 | 拟解决的主要技术挑战 |
| 目标 IPC 分类号 | 如已知，直接使用；如未知，由 Skill 推荐 |
| 检索时间范围 | 默认近10年，核心领域可聚焦近5年 |
| 地域范围 | 中、美、欧、日、韩、PCT（默认全部） |
| 企业/机构名称 | 用于模块2企业专利组合分析 |
| 技术方案描述 | 用于模块3查新与创新点论证 |

### 1.2 检索策略制定

根据收集到的信息，构建如下检索配置：

```plain
检索式 = 技术关键词组 AND/OR 功效关键词组 AND/OR 应用场景词组
IPC 范围 = [基于技术领域推导]
法律状态 = 审中 + 有效（竞争格局分析）/ 仅有效（FTO分析）
专利类型 = 优先发明专利
时间范围 = 近10年（近5年重点分析）
```

### 1.3 检索执行（调用智慧芽 MCP）

**检索工具调用规则**：

1. **趋势分析** → 调用 `mcp_patsnap-patent-technology-landscape__trends`
   - 参数：`keywords`（技术关键词）、`apply_start_time`（起始年）、`apply_end_time`（截止年）、`lang="cn"`
   
2. **申请人排名** → 调用 `mcp_patsnap-patent-technology-landscape__applicant_ranking`
   - 参数：`keywords`（技术关键词）、`lang="cn"`

3. **发明人排名** → 调用 `mcp_patsnap-patent-technology-landscape__inventor_ranking`
   - 参数：`keywords`（技术关键词）、`lang="cn"`

4. **地域分布** → 调用 `mcp_patsnap-patent-technology-landscape__priority_country`
   - 参数：`keywords`、`lang="cn"`

5. **技术词云/热点** → 调用 `mcp_patsnap-patent-technology-landscape__word_cloud`
   - 参数：`keywords`、`lang="cn"`

6. **创新词云（更细分热点）** → 调用 `mcp_patsnap-patent-technology-landscape__wheel_of_innovation`
   - 参数：`keywords`、`lang="cn"`

7. **法律状态分布** → 调用 `mcp_patsnap-patent-technology-landscape__simple_legal_status`
   - 参数：`keywords`、`lang="cn"`

8. **专利价值分布** → 调用 `mcp_patsnap-patent-technology-landscape__portfolio_value`
   - 参数：`keywords`、`lang="cn"`

9. **被引最多专利（核心技术识别）** → 调用 `mcp_patsnap-patent-technology-landscape__most_cited`
   - 参数：`keywords`、`lang="cn"`

10. **诉讼风险专利** → 调用 `mcp_patsnap-patent-technology-landscape__most_asserted`
    - 参数：`keywords`、`lang="cn"`

11. **企业专利组合概览** → 调用 `mcp_patsnap-patent-technology-landscape__company_portfolio_overview`
    - 参数：`company_name`（用户企业名称）、`lang="cn"`

12. **企业技术构成** → 调用 `mcp_patsnap-patent-technology-landscape__company_key_technology`
    - 参数：`company_name`、`lang="cn"`

13. **企业申请趋势** → 调用 `mcp_patsnap-patent-technology-landscape__company_patent_trends`
    - 参数：`company_name`、`lang="cn"`

14. **企业被引最多专利** → 调用 `mcp_patsnap-patent-technology-landscape__company_most_cited_latest`
    - 参数：`company_name`、`lang="cn"`

15. **企业法律状态** → 调用 `mcp_patsnap-patent-technology-landscape__company_simple_legal_status`
    - 参数：`company_name`、`lang="cn"`

16. **企业创新战略雷达** → 调用 `mcp_patsnap-patent-technology-landscape__company_strategy_map`
    - 参数：`company_name`、`lang="cn"`

17. **专利精准检索（查新/FTO）** → 调用 `mcp_patsnap-valuation__search_patents`
    - 参数：`query_text`（构建检索式）、`limit=20`

18. **专利语义搜索（相关技术方案）** → 调用 `mcp_patent-search__patsnap_search`
    - 参数：`search_strategy=["semantic","keyword"]`、`semantic_query`、`keywords`、`sources=["patent"]`、`topk=20`

19. **专利全文获取（FTO用）** → 调用 `mcp_patent-search__patsnap_fetch`
    - 参数：`keys`（专利号列表）、`key_type="pn"`、`module=["basic"]`

20. **单件专利价值评估** → 调用 `mcp_patsnap-patent-value-assessment__valuation`（或 `mcp_patsnap-valuation__valuation_get_patent_valuation`）
    - 参数：`patent_number`

**检索记录模板**（每次检索后记录）：
```plain
检索批次：[N]
检索目标：[模块X - 分析项]
检索工具：[工具名]
检索式/参数：[具体参数]
命中数量：[XX] 件
检索时间：[YYYY-MM-DD]
```

## Step 2：7大模块分析执行框架

### 模块一：国内外研究现状与竞争格局

**数据调用**：
- 趋势：`trends` → 近10-20年申请量/授权量年度数据
- 申请人：`applicant_ranking` → Top10申请人
- 地域：`priority_country` → 各国/地区占比
- 热点：`word_cloud` + `wheel_of_innovation` → 技术热词
- 价值：`portfolio_value` → 专利价值分布
- 法律状态：`simple_legal_status` → 有效/审中/失效占比

**分析维度**：

| 分析项 | 调用工具 | 输出结论 |
|--------|----------|----------|
| 技术生命周期 | `trends` | 萌芽期/成长期/成熟期/衰退期 |
| 主要竞争主体 | `applicant_ranking` | 竞争对手清单及专利规模 |
| 技术热点分布 | `word_cloud` / `wheel_of_innovation` | 技术分支热度图谱 |
| 地域布局态势 | `priority_country` | 技术主导国/市场布局策略 |
| 专利活跃度 | `simple_legal_status` | 有效专利占比/技术壁垒强度 |
| 核心基础专利 | `most_cited` | 行业基础专利及持有人 |

**立项文件输出模板**：

```markdown
### 1.X 国内外研究现状与竞争格局（基于专利分析）

经对 PatSnap 智慧芽专利数据库检索，检索式为"[检索式]"，检索时间范围 [YYYY]-[YYYY] 年，
共命中 [XX] 件专利（发明专利 [XX] 件，PCT [XX] 件），分析结论如下：

**（1）技术发展趋势**：近5年全球专利申请量年均增长 [XX]%，其中中国占比 [XX]%，
美国占比 [XX]%。该技术领域目前处于[成长期/成熟期]，近2年申请量[持续上升/趋于平稳/开始下降]。

**（2）主要竞争主体**：全球专利布局前5位分别为 A公司（[XX]件，核心IPC：XX）、
B公司（[XX]件）、C大学（[XX]件）...。国内主要申请人为...。

**（3）技术热点与空白**：通过关键词聚类分析，[XX]技术分支专利密度最高（[XX]件），
属于红海竞争区；而[XX]技术分支全球布局不足[XX]件，存在显著技术空白。

**（4）地域布局态势**：该技术领域专利以[中国/美国]为首要布局地，欧洲、日本次之。
头部企业普遍采用[中国+美国+PCT]的全球化布局策略。

**（5）专利活跃度**：当前有效专利占比 [XX]%，审中专利 [XX]%，表明该领域
[技术创新持续活跃/技术格局趋于固化]。

**（6）核心基础专利**：被引次数最高的专利为 [专利号]（[申请人]，被引 [XX] 次），
保护的核心技术为 [XX]，是该领域的奠基性专利。
```

### 模块三：创新点与先进性论证

**数据调用**：
- `mcp_patent-search__patsnap_search`（语义检索）→ 最相关专利TOP20
- `mcp_patsnap-valuation__search_patents`（关键词精准检索）→ 查新结果
- `mcp_patent-search__patsnap_fetch`（获取最相近专利全文）→ 详细对比

**分析维度**：

| 分析项 | 调用工具 | 输出结论 |
|--------|----------|----------|
| 查新结论 | `patsnap_search` + `search_patents` | 拟采用方案是否已被公开 |
| 最接近现有技术 | `patsnap_fetch` 获取全文 | 差异点对比 |
| 先进性参数对标 | 语义检索结果提取 | 领先幅度量化 |

**立项文件输出模板**：

```markdown
### 3.X 创新点与先进性论证（基于专利分析）

**（1）查新结论**：经对 PatSnap 智慧芽数据库检索（检索式：[XX]，命中 [XX] 件），
本项目拟采用的"[具体技术方案描述]"尚未被现有专利完整公开。

最接近的现有技术为 [专利号]（[申请人]，申请日：[YYYY-MM-DD]），其技术方案为 [XX]，
与本项目的核心区别在于：[区别技术特征1]、[区别技术特征2]。

**（2）技术功效矩阵分析**：对检索到的 [XX] 件核心专利进行技术-功效矩阵分析，
现有技术主要集中在["高效率-高成本"和"低效率-低成本"]象限，
而["高效率-低成本"]象限尚无专利布局。本项目拟突破的技术路线恰好填补该空白。

| 技术手段＼解决问题 | 提高效率 | 降低成本 | 提高可靠性 |
|------------------|--------|---------|-----------|
| 主流方案A | ★★★ | ★ | ★★ |
| 次主流方案B | ★★ | ★★ | ★ |
| **本项目方案** | ★★★ | ★★★ | ★★★ |

**（3）先进性对标**：本项目目标将 [XX指标] 提升至 [XX]，而现有专利公开的最佳水平为
[XX]（[专利号]，[申请人]），本项目预期领先幅度约 [XX]%。
```

### 模块四：知识产权布局规划

**数据调用**：
- `applicant_ranking` → 主要竞争对手及专利规模
- `word_cloud` + `wheel_of_innovation` → 技术热点与空白
- `simple_legal_status` → 当前有效专利格局
- `company_key_technology` → 我方技术覆盖盲区识别

**分析维度**：

| 分析项 | 调用工具 | 输出结论 |
|--------|----------|----------|
| 布局空白点 | 竞争格局数据 → 盲区识别 | 待布局技术分支/应用场景 |
| 防御性布局 | `most_cited` → 周边专利 | 外围专利布局节点 |
| 进攻性布局 | 申请人分布薄弱环节 | 卡位布局方向 |
| 地域策略 | `priority_country` 空白地区 | 优先布局国家/地区 |

**立项文件输出模板**：

```markdown
### 4.X 知识产权布局规划

**（1）布局空白点**：基于竞争对手专利地图分析，在 [XX技术分支] 方向全球仅有 [XX] 件专利，
竞争对手布局稀疏。本项目将重点围绕以下空白点布局：
- 空白点1：[技术分支/应用场景]，建议申请发明专利
- 空白点2：[技术分支/应用场景]，建议申请发明专利

**（2）防御性布局**：针对行业核心专利 [专利号]（[申请人]，保护 [XX]），
本项目将在其周边布局改进型外围专利，降低侵权风险，增强交叉许可筹码。

**（3）进攻性布局**：[XX技术方向]当前专利多由高校/研究机构持有，商业化部署薄弱，
本项目可提前卡位，争取形成该方向的商业化专利主导权。

**（4）地域布局策略**：优先在中国（核心市场）、美国（主要竞争市场）布局，
结合 PCT 申请覆盖欧洲/日本/韩国目标市场。

**（5）布局节奏**：

| 阶段 | 时间 | 专利申请目标 | 重点方向 |
|-----|------|------------|---------|
| 第一阶段 | 第1年 | 发明专利申请 | 核心技术方案保护 |
| 第二阶段 | 第2年 | 发明专利申请 | 外围改进/应用场景扩展 |
| 第三阶段 | 第3年 | 发明专利+PCT申请 | 国际布局 |
```

### 模块五：技术路线选择验证

**数据调用**：

- `simple_legal_status`（分路线）→ 各路线壁垒强度
- `most_cited`（分路线）→ 各路线核心专利

**分析维度**：

| 分析项 | 调用工具 | 输出结论 |
|--------|----------|----------|
| 主流路线验证 | `trends`（分路线） | 行业主流技术路线 |
| 路线专利壁垒 | `applicant_ranking` + `most_cited` | 进入壁垒高低 |
| 路线活跃度 | 近3年新增专利占比 | 路线热度与技术成熟度 |

**立项文件输出模板**：

```markdown
### 5.X 技术路线专利验证

**（1）主流路线识别**：经分路线专利检索，当前行业存在 [路线A]、[路线B]、[路线C] 三条
主要技术路线，近5年专利产出分别为 [XX] 件、[XX] 件、[XX] 件，
[路线X] 为行业主流（占比 [XX]%）。

**（2）路线壁垒评估**：

- **路线C（本项目选择）**：近3年增速 [XX]%，核心专利多由高校持有，
  商业化壁垒相对较低，存在差异化发展空间。

**（3）结论**：综合专利态势，本项目选择 [XX技术路线] 具有差异化优势，
专利风险可控，技术发展空间充足，建议按此路线推进。
```

### 模块六：知识产权风险分析（FTO）

**数据调用**：
- `mcp_patsnap-valuation__search_patents`（精准检索有效专利）→ 风险专利清单
- `mcp_patent-search__patsnap_fetch`（获取高风险专利全文）→ 权利要求比对
- `most_asserted`（技术领域）→ 诉讼最多的高风险专利
- `mcp_patsnap-valuation__legal_status`（具体专利）→ 确认法律状态
- `mcp_patsnap-valuation__claims`（具体专利）→ 权利要求文本

**分析维度**：

| 分析项 | 调用工具 | 输出结论 |
|--------|----------|----------|
| 侵权风险识别 | `search_patents`（有效专利）→ 特征对比 | 高/中/低风险判定 |

## 执行注意事项

1. **渐进式交互**：如用户未提供完整信息，先执行已知部分，同步询问缺失项，不得卡住等待全部信息
2. **数据真实性**：所有数量、专利号、申请人均来自 MCP 工具返回的真实数据，禁止捏造
3. **法律声明**：FTO 分析章节必须包含"仅供参考，正式意见需执业律师出具"声明
4. **时效声明**：报告必须注明检索日期，说明法律状态数据时效性
5. **禁止出现费用相关内容**：包括代理费、官方费、年费等任何费用信息
6. **语言适配**：根据用户提问语言自动切换，专利号保留原始格式
7. **并行检索**：Step 2 各模块中互相独立的 MCP 工具调用尽量并行执行，提高效率
