# patent-infringement-watch

## 功能概述

竞争对手产品专利侵权监测工具，支持两条工作路径：

- **路径 A（文件上传模式）**：用户提供己方公司名称 + 上传竞对产品详情文件（PDF/Word/TXT），自动提取产品技术特征，匹配己方高相关专利，逐要素比对分析侵权风险。
- **路径 B（竞对公司名模式）**：用户提供己方公司名称 + 竞对公司名称，自动通过公开渠道（网页、专利申请、新闻等）收集竞对产品情报，再执行与路径 A 相同的匹配与比对流程。

## 触发方式

以下任意情形均应加载本 skill：
- 用户说"监测竞争对手产品是否侵权"
- 用户说"分析竞对产品是否落入我方专利"
- 用户上传竞对产品资料并询问是否被我方专利覆盖
- 用户输入竞对公司名并要求进行专利比对
- 用户要求生成侵权分析报告 / FTO 反向分析

## 工作流程（7步）

```
Step 1  收集输入
        ├─ 路径A: 己方公司名 + 竞对产品文件（PDF/Word/TXT/图片）
        └─ 路径B: 己方公司名 + 竞对公司名 [+ 可选产品线关键词]

Step 2  检索己方专利组合
        ├─ 调用 patent.search，filter: assignees=[己方公司名]
        ├─ 获取有效专利列表（legal_status=active 优先）
        └─ 按 IPC 分类分组，标注独立权利要求数量

Step 3  获取竞对产品技术特征
        ├─ 路径A: 解析上传文件，提取技术特征清单
        └─ 路径B:
            ├─ web.search 搜索竞对公司官网、产品页、发布会
            ├─ web.search 搜索竞对公司专利申请（揭示产品实现）
            ├─ web.search 搜索行业测评、拆解报告
            └─ 综合提炼竞对产品技术特征清单

Step 4  语义相关性匹配
        ├─ 以竞对产品技术特征为 query，语义检索己方专利
        ├─ 筛选相关度 Top-N 专利
        └─ 对 Top 专利调用 patent.fetch 获取权利要求全文

Step 5  权利要求要素比对
        ├─ 逐条拆解独立权利要求必要技术特征
        ├─ 字面侵权比对（全面覆盖原则）
        ├─ 等同侵权比对（Function-Way-Result）
        └─ 标注每个要素：✅ 字面覆盖 / ⚠️ 等同可能 / ❌ 缺失

Step 6  风险评级
        ├─ 高风险：所有独立权利要求要素均字面覆盖
        ├─ 中风险：部分要素等同覆盖，整体读入可能
        └─ 低风险：关键要素缺失，当前版本不落入

Step 7  输出侵权分析报告
        ├─ 执行摘要（风险等级 + 核心结论）
        ├─ 己方专利组合概况
        ├─ 竞对产品技术特征清单（含来源）
        ├─ 逐专利要素比对矩阵
        ├─ 等同原则分析（如适用）
        ├─ 风险等级评定表
        └─ 建议行动方案
```

## 输入规格

| 参数 | 路径A | 路径B |
|------|-------|-------|
| 己方公司名称 | 必填 | 必填 |
| 竞对产品文件 | 必填（PDF/Word/TXT） | 可选补充 |
| 竞对公司名称 | 可选 | 必填 |
| 产品线关键词 | 可选 | 可选（缩窄搜索范围） |
| 目标专利号 | 可选（指定比对专利） | 可选（指定比对专利） |
| 分析管辖域 | 可选（CN/US/EP，默认CN） | 可选 |

## 输出规格

- **执行摘要**：风险等级（高/中/低）+ 触发专利数量 + 一句话结论
- **专利组合表**：专利号、标题、独立权利要求数、法律状态、到期日
- **产品特征清单**：编号特征列表 + 信息来源引用
- **比对矩阵**：每条独立权利要求 × 产品特征要素的逐项比对
- **等同原则分析**：Function / Way / Result 三要素论述（如有等同可能）
- **风险评定表**：每件专利的综合风险等级 + 评定依据
- **行动建议**：按优先级排列的后续行动清单

## 分析法律标准

- **中国**（默认）：全面覆盖原则（《专利法》第64条）+ 等同原则（最高院司法解释第7条）+ 禁止反悔
- **美国**：All-Elements Rule + Doctrine of Equivalents (Graver Tank) + File Wrapper Estoppel
- **欧洲**：Protocol on Interpretation of Article 69 EPC + Equivalents

## 重要声明

本 skill 输出结果仅供内部参考，不构成正式法律意见。高风险结论须委托持牌专利律师或专利代理机构进行独立复核后方可用于诉讼或行动决策。

## 工具依赖

- `patent.search` → `mcp_patent-search__patsnap_search`：检索己方专利组合
- `patent.fetch` → `mcp_patent-search__patsnap_fetch`：获取权利要求全文
- `web.search` → `mcp_web-search__web_search`：收集竞对产品公开情报
- `web_fetch`：抓取竞对产品网页详情
- `files`：解析上传的竞对产品文件

## 注意事项

- 专利检索以 `legal_status=active` 为优先，失效专利仅供参考
- 竞对产品特征提取须引用可追溯的公开来源，不得主观推断
- 等同原则比对须明确说明"方式相同、功能相同、效果相同"的具体依据
- 禁止反悔检查：若专利有审查历史限缩，须在比对结论中特别标注
- 所有比对结论须区分「字面覆盖」与「等同可能」，不得混淆
