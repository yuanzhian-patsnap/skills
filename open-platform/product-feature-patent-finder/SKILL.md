# Product Feature Patent Finder

## 用途

输入产品特性描述，自动完成以下三步工作流，输出一份 HTML 报告：

1. **产品特性调研**：对产品特性描述进行网络调研，整理出带编号的关键技术拆解维度列表
2. **专利检索、评分与技术维度对照**：根据关键技术描述检索相关专利清单，给出相关性评分（0–10）、相关性解读，并标注每条专利覆盖的技术拆解维度
3. **HTML 报告生成**：将以上内容整合输出为结构化的 HTML 报告，含维度筛选交互

## 输入

- `product_feature`：产品特性描述（自然语言，中英文均可，建议 50–2000 字）

可选参数：
- `assignee`：限定专利权人/申请人（如 "Apple"、"华为"），留空则不限
- `jurisdiction`：限定专利受理局（如 "CN"、"US"），留空则不限
- `date_from` / `date_to`：专利申请日期范围（格式 YYYYMMDD）
- `top_n`：返回专利数量上限，默认 20

## 输出

一份自包含 HTML 文件，路径为 `@session/reports/patent_finder_<timestamp>.html`，包含：

### Part 1：产品特性概览
- 产品特性描述（用户原文）
- 关键技术拆解（带编号 T1–TN 的技术维度卡片，每张卡片显示维度编号、名称、简要说明）
- 产品公开资料摘要（来自网络调研，附链接）
- 产品图片（如可获取则嵌入，否则显示来源链接）

### Part 2：相关专利清单
- 专利号（附智慧芽链接）
- 专利标题
- 申请人 / 专利权人
- 法律状态
- 公开日
- **相关性评分**（0–10，彩色徽章）
- **相关性解读**（1–3 句话）
- **对应技术维度**（彩色 tag 徽章，如 `T1` `T3` `T5`，对应 Part 1 的技术拆解编号）

报告顶部设置**技术维度筛选器**：点击 T1–TN 按钮可高亮/过滤显示覆盖该维度的专利。

## 工作流（Step-by-Step SOP）

### Step 1：产品特性调研 + 技术拆解建模

1. 解析用户输入的 `product_feature`，识别核心功能点
2. 使用 `web.search` 检索产品公开资料、官方介绍、技术白皮书等
3. 从检索结果中提取：
   - 产品功能描述（1–2 段）
   - **关键技术拆解维度列表**（3–8 条，每条赋予编号 T1、T2…TN）：
     - 每个维度聚焦一个独立技术层面（感知层、识别层、融合层、交互层等）
     - 写明维度名称（中英文）+ 一句技术说明
     - 示例：`T1 眼动追踪（Eye/Gaze Tracking）：通过红外传感器实时捕获用户注视点`
   - 产品图片 URL（如有）
4. 记录所有信息来源链接

> **注意**：技术拆解维度是后续专利对照的核心索引，维度划分需互斥、完整，覆盖产品特性的完整技术链路。

### Step 2：专利检索、评分与技术维度对照

1. 基于 Step 1 的关键技术描述，构建 2–4 组检索策略：
   - **语义检索**（`search_strategy: ["semantic"]`）：对每条关键技术描述进行语义搜索
   - **关键词检索**（`search_strategy: ["keyword"]`）：提取技术术语进行关键词搜索
   - 若用户指定了 `assignee` / `jurisdiction` 等参数，追加 `filter` 策略
2. 合并所有检索结果，去重，取 `top_n` 条（默认 20）
3. 对每条专利进行相关性评分（0–10）：

   | 分数 | 含义 |
   |------|------|
   | 9–10 | 高度相关：专利核心技术与产品特性直接对应 |
   | 7–8  | 相关：专利涵盖产品特性的关键技术手段之一 |
   | 5–6  | 部分相关：专利涉及相关技术领域但不直接覆盖 |
   | 3–4  | 弱相关：技术背景有关联，但核心技术有差异 |
   | 0–2  | 不相关：仅因关键词偶合命中，技术无实质关联 |

4. **技术维度对照**（核心新步骤）：
   - 对照 Step 1 建立的 T1–TN 技术拆解维度，逐条判断每件专利覆盖了哪些维度
   - 字段名：`tech_dimensions`，格式为编号列表，如 `[T1, T3]`
   - 对照规则：
     - 若专利的权利要求/摘要直接描述了该维度的技术手段 → 标注该维度
     - 若专利仅属于相关技术背景而未直接覆盖该维度 → 不标注
     - 一条专利可覆盖多个维度（如同时涉及 T1 眼动追踪和 T4 凝视触发）
     - 评分 0–2 的专利不强制标注维度
5. 撰写每条专利的相关性解读（1–3 句话，说明与哪些维度的技术关联）

### Step 3：HTML 报告生成

按照以下结构输出 HTML 报告：

```
[封面] 报告标题 / 生成日期 / 输入摘要
[Part 1] 产品特性概览
  - 特性描述
  - 关键技术拆解（技术点卡片，每张卡片显示 Tx 编号 + 名称 + 说明）
  - 产品资料摘要（含图片或链接）
[Part 2] 专利清单
  - 顶部：技术维度筛选器（T1–TN 按钮，点击高亮/过滤）
  - 按相关性评分降序排列
  - 每条专利展示：专利号链接、标题、申请人、状态、公开日、评分徽章、技术维度标签、解读
[来源] 网络调研来源列表
```

HTML 要求：
- 自包含（CSS 内嵌，无外部依赖 JS 框架）
- 响应式布局（桌面 + 移动友好）
- 专利号可点击跳转至智慧芽页面（`https://eureka.zhihuiya.com/view/#/fullText?patentId=...`，patentId 从检索结果获取）
- 相关性评分用彩色徽章展示（9–10：绿色；7–8：蓝色；5–6：橙色；3–4：灰色；0–2：红色）
- **技术维度标签**：每个 Tx 用独立彩色圆角徽章展示，颜色与 Part 1 技术拆解卡片一一对应（T1 固定一种颜色，贯穿全报告），颜色调色板：T1=#3B82F6、T2=#10B981、T3=#F59E0B、T4=#EF4444、T5=#8B5CF6、T6=#06B6D4、T7=#F97316、T8=#EC4899，超过 8 个维度循环使用
- **维度筛选器**（纯 JS 实现，无外部依赖）：
  - 顶部展示 T1–TN 筛选按钮 + "全部显示"按钮
  - 点击某维度按钮：仅显示含该维度标签的专利行，其余隐藏
  - 再次点击或点击"全部"：恢复显示所有专利
  - 按钮激活状态有视觉高亮（边框加粗/背景变深）
- 大文件使用 `begin_write` + `append` + `finish_write` 分块写入，不得在单次 content 参数中嵌入完整 HTML

## 工具调用规范

| 步骤 | 工具 | 说明 |
|------|------|------|
| Step 1 调研 | `web.search` → `web_fetch` | 检索产品资料并抓取页面内容 |
| Step 2 专利检索 | `patent.search`（语义 + 关键词）| 多轮检索，合并去重 |
| Step 2 评分 + 维度对照 | AI 推理（基于摘要 + 技术拆解对比） | 不调用外部工具 |
| Step 3 报告 | `files.begin_write` + `files.append` × N + `files.finish_write` | 分块写入 HTML |

## 注意事项

- **不得捏造**专利号、引用次数、法律状态等数据，所有专利信息必须来自 `patent.search` 工具返回结果
- 技术维度标注需基于专利摘要/权利要求的实际内容，不得随意标注
- 产品图片若无法获取，显示占位符并标注来源，不得使用随机图片
- 相关性评分需基于技术对比推理，不得随意打分
- HTML 文件较大时，必须分块写入（每块约 6000–8000 字符），禁止单次大块写入
- 如用户未提供 `assignee` 等可选参数，检索时不加限制，覆盖全球专利
