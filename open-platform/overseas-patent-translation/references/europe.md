# Europe Patent Translation Reference

## Target Language

Use English unless the user requests French or German. Format for EPO-style filing and prosecution.

## Drafting Focus

- Emphasize clarity, support, and technical character.
- Avoid result-only language where concrete technical features are available.
- Keep amendment basis visible by preserving layered embodiments and fallback positions.
- Avoid adding technical effects not expressly or implicitly supported by the Chinese priority text.
- Translate functional features only when the source supports the function and the description provides means for implementation.

## Recommended Section Order

```text
Title
Description
  Technical Field
  Background Art
  Summary
  Brief Description of Drawings
  Detailed Description of Embodiments
Claims
Abstract
Drawings / Reference Signs
Sequence Listing, if any
```

## Claim Style Notes

- Use "comprising" for "包括/包含" unless a closed scope is clearly intended.
- Prefer clear antecedent basis: "the processor", "the first member", "the data packet".
- Avoid unnecessary "means for" phrasing.
- Keep claim categories distinct: product/device/system/method/computer-readable medium.
- For method claims, preserve only necessary step order.

## QA Focus

- Check that every claim feature has description support.
- Check that reference signs are optional aids and do not narrow the claim text.
- Check that problem/effect statements are not stronger than source support.
