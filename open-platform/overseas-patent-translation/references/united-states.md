# United States Patent Translation Reference

## Target Language

Use English. Format for USPTO utility application drafting.

## Drafting Focus

- Preserve broad but supported claim scope.
- Avoid unnecessary admissions in Background.
- Avoid unsupported superlatives or absolute effect statements.
- Support claim construction by keeping definitions, alternatives, examples, and implementation details clear.
- Watch possible means-plus-function risk when translating "模块", "单元", "装置", or "用于...的部件".

## Recommended Section Order

```text
Title of Invention
Cross-Reference to Related Applications, if applicable
Technical Field
Background
Summary
Brief Description of the Drawings
Detailed Description
Claims
Abstract
Drawings
Sequence Listing, if any
```

## Claim Style Notes

- Use "comprising" for open-ended "包括/包含".
- Use "at least one" carefully for "至少一个"; keep singular/plural consistency.
- Avoid "consisting of" unless the Chinese text and filing strategy require closed scope.
- Keep antecedent basis clear with "a/an" on first introduction and "the" later.
- Translate "连接", "耦合", "电连接", "通信连接", and "固定连接" according to the actual technical relation.
- For method claims, do not add mandatory order unless source language or technical logic requires it.

## QA Focus

- Check for accidental narrowing by "must", "only", "all", "always", or rigid sequence language.
- Check that "module/unit" translations do not create avoidable functional-claim ambiguity.
- Check Background for statements that may be read as applicant admissions.
