# 追溯文档模板

每次生成 HTML 报告后，必须输出一份 Markdown 追溯文档，记录完整的检索、分析、生成过程。

**文件命名：** `~/Documents/tech_briefings/tech_briefing_v2_<YYYYMMDD_HHMMSS>_trace.md`（与 HTML 同名 + `_trace`）

## 编写原则

1. **如实记录** — 不美化、不省略，包括失败和重试
2. **可追溯** — 每个数据点能追溯到具体 API 调用和返回结果
3. **可验证** — 读者可在智慧芽中复现检索
4. **标注决策** — 对于主观判断（分类、相关性），说明依据
