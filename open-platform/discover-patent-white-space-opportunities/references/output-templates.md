# 输出模板

## 一、阶段成果

### 阶段结论
用一个简洁段落说明本阶段结论及其意义。

| 项目 | 内容 |
|------|------|
| 检查内容 | 使用的范围、数据和参照 |
| 支持证据 | 支持结论的最强证据 |
| 反方证据 | 与结论相反的证据或其他解释 |
| 推理过程 | 证据如何支持阶段结论 |
| 置信度 | 高、中或低，并说明理由 |
| 未知项 | 可能改变结论的缺失证据 |
| 下一项动作 | 下一步分析动作或用户需要作出的决定 |

## 二、候选空白点表

| 候选空白点 | 实际专利数量 | 选择理由 | 可能的问题 | 假空白风险 | 优先级 | 置信度 |
|-----------|------------|---------|-----------|-----------|-------|-------|

表格后推荐一个候选项，并启动强制确认点1。

> ⚠️ 候选表不得出现"预计数量""预期数量""实际为预期的比例"等推算字段。

**强制确认点1话术示例：**
> 我识别出以上候选空白点。建议优先分析【候选项】，因为【理由】。请确认希望深入分析哪一个；也可以指定多个，我会分别论证。

🔴 **停止并等待用户明确确认，确认前不得进入深度分析。**

| 替代方案不足 | | | |
| 行业解决动力 | | | |
| 未来趋势 | | | |

必须包括：
- 支持问题价值的证据；
- 反对问题价值的证据；
- 暂定结论；
- 最大不确定性；
- 强制确认点2的问题。

**强制确认点2话术示例：**
> 当前判断是：【问题】具有【高/中/低】价值，主要依据是【依据】，最大不确定性是【不确定性】。请确认这一判断是否符合您的行业认知，以及是否继续进入矛盾诊断和化解思路生成。
