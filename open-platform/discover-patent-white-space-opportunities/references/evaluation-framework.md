# 候选空白点评估框架

本文件供阶段1候选空白点初步排序，以及阶段3问题价值评估时使用。

---

## 一、候选空白点初步排序

对每个候选空白点，按以下六个维度评分，每项 1–5 分，并展示评分证据。

| 维度 | 核心问题 |
|------|----------|
| 稀疏程度 | 该交叉点的实际专利数量是否较低？ |
| 两侧吸引力 | 构成空白的技术、功效、应用或路线是否分别活跃且重要？ |
| 需求信号 | 是否存在重要且尚未满足的问题证据？ |
| 矛盾线索 | 是否存在现有路线难以兼顾的目标或断裂环节？ |
| 战略相关性 | 是否符合用户的市场、能力和决策目标？ |
| 假空白风险 | 分类、检索、阻断专利或商业秘密是否可能解释空白？（分值越高代表风险越低） |

> **注意：** 评分仅用于候选项排序，不可据此直接宣布创新机会成立。不得通过行列数量推算交叉点的预计专利数量或异常百分比。

---

## 二、空白的初步解释类型

在展示候选空白点时，对每项给出初步解释类型：

| 类型 | 含义 |
|------|------|
| 结构性矛盾空白 | 问题重要，但现有路线存在难以兼顾的冲突 |
| 路线断裂空白 | 相关技术路线分别存在，但缺少有效连接方式 |
| 新兴组合空白 | 相邻技术已经成熟，但尚未形成目标组合 |
| 暂不可解空白 | 问题重要，但当前缺少明显的化解条件 |
| 需求不足空白 | 专利稀疏是因为问题价值不足 |
| 表观空白 | 由分类、术语、数据或检索方式造成 |
| 隐蔽活动空白 | 相关活动可能通过商业秘密保护，未体现于专利 |

---

## 三、问题价值评估

用于阶段3，对用户确认后的候选空白点所对应的问题进行价值评估。

| 维度 | 较弱表现（→低分） | 较强表现（→高分） |
|------|-----------------|-----------------|
| 严重程度 | 轻微不便 | 重大成本、安全、合规或战略后果 |
| 发生频率 | 少见边缘情况 | 高频或系统性问题 |
| 影响范围 | 狭窄细分群体 | 大规模或快速增长群体 |
| 当前替代方案 | 有效且成本合理 | 无效、昂贵、缓慢或风险较高 |
| 解决动力 | 缺少行动意愿 | 存在明确法规、运营或战略压力 |
| 未来趋势 | 重要性下降 | 紧迫性或市场规模持续增长 |

> 仅当问题具有较高价值，或用户明确选择继续探索时，才进入矛盾诊断。

---

## 四、矛盾诊断检查表

在阶段4诊断空白成因与关键矛盾时，逐项检查：

1. 现有路线是否分别优化不同目标？
2. 改善一个目标时，是否会导致另一个目标恶化？
3. 是否存在两条互不连接的解决路线？
4. 路线之间具体缺少什么连接、转换或反馈机制？
5. 哪一个矛盾最能解释该空白，应被定义为主要矛盾？
6. 还有哪些影响解决效果的次要矛盾？
7. 关键冲突是技术性的，还是来自成本、资源、法规或生态约束？
8. 空白是否可以由一个清晰的因果机制解释？
9. 化解思路是否直接作用于该因果机制？

---

## 五、置信度标准

| 等级 | 标准 |
|------|------|
| 高 | 获得多个独立来源和直接证据支持 |
| 中 | 判断合理且有证据支持，但仍存在重要假设 |
| 低 | 主要依赖推断，或关键数据缺失 |

> 始终说明哪项新增证据最可能改变当前结论。

---

## 六、注意事项

- 所有评分均为辅助工具，最终判断须结合用户的行业专家知识。
- 不得仅凭专利数量判断问题价值；专利多少是地图信号，不是问题重要性的直接指标。
- 置信度须明确标注，尤其当底层数据不足时。
- 候选表中禁止出现"预期专利数量"或"实际/预期比率"等推算指标。
