> **版本：V5（授权导向专业版）**

---

## 一、Skill 定位与触发条件

### 适用场景
- 用户提供一件专利（号码/文本）或技术交底书，要求做全周期或某阶段专利工作
- 用户需要从授权专利挖掘后续创新点并批量申请
- 用户需要查新报告、权利要求草稿、OA答复建议、侵权风险评估等专业输出
- 用户需要了解某件专利的商业价值、维持决策或许可策略
- 用户需要生成技术交底书（含发明目的/实施例/附图说明/权利要求建议）
- 用户需要生成专利结构附图（PNG工程图 + DXF CAD文件，Base64内嵌HTML）
- 用户要求"以下证授权为目标"→ 强制触发**授权导向模式**

### 不适用场景
- 纯市场/竞争战略分析（用 tech-strategy-analysis）
- 纯行业技术动态周报（用 chanye-zhoubao）
- 非专利类知识产权（商标、著作权等）

## 三、全周期工作流（七阶段）

```
【前期】
阶段1  技术交底与战略规划   → 技术地图 + 布局策略 + 申请时机建议
阶段2  FTO & 查新检索       → 新颖性判断 + 自由实施风险 + 查新报告

【中期】
阶段3  专利文件撰写辅助     → 权利要求草稿 + 说明书框架 + 实施例建议
阶段3b 技术交底书撰写       → 完整交底书（含发明目的/实施例/附图/权利要求建议）
阶段3c 专利附图生成         → PNG工程图（Base64内嵌）+ DXF CAD文件
阶段4  提交与审查应对       → OA答复策略 + 修改方案 + 驳回救济建议

【后期】
阶段5  授权确认与登记       → 授权文本核查 + 费用节点提醒
阶段6  权利维护与运营       → 年费管理 + 价值评估 + 许可/转让建议
阶段7  侵权监控与防御       → 侵权预警 + 无效策略 + 诉讼路径建议
```

## 四、V5授权导向模式（GRANT_MODE=on，默认强制执行）

> ⚡ V5核心新增：凡涉及专利申请文件生成，必须执行以下9项授权导向专业检查，不得跳过。

### 9项授权导向强制项

| # | 检查项 | 执行标准 |
|---|--------|---------|
| 1 | **独权区别特征精准化** | 独立权利要求必须明确写入与最近似现有技术的区别特征，功能性限定语连接各技术特征 |
| 2 | **功能性备用权利要求** | 每件增加1条功能性权利要求作为OA答复修改预案 |
| 3 | **方法权利要求步骤对齐** | 方法权利要求步骤顺序与说明书工作原理章节严格一致，检测→判断→执行→反馈四步完整 |
| 4 | **最宽实施方式预埋** | 说明书中增加"本发明不限于上述具体实施方式，凡采用……原理实现……功能的方案均属于本发明保护范围"段落 |
| 5 | **对比实验数据** | 实施例中加入至少1组对比例，与现有技术定量对比，数据来源标注（实验测试/理论计算） |
| 6 | **背景技术批评精准化** | 背景技术缺陷描述与发明目的一一对应，并指向具体对比文献（含专利号） |
| 7 | **OA答复预案写入** | 每件专利预判2条最可能审查意见，写入预案（修改策略+争辩要点） |
| 8 | **IPC分类号建议** | 每件给出主IPC+副IPC建议（2～3个） |
| 9 | **附图数量充分** | 每件附图不少于4张（含整体结构图/截面图/时序图/特性曲线图） |

### 授权把握度评估标准

| 综合评分 | 授权把握度 | 处理建议 |
|---------|-----------|---------|
| 97～100分 | 🟢 92%+ | 可直接提交 |
| 90～96分 | 🟡 80～91% | 建议补全弱项后提交 |
| 低于90分 | 🔴 <80% | 必须修复后方可提交 |

## 五、各阶段执行细节

### 阶段1：技术交底与战略规划

**输入：** 技术交底书文本 或 已有专利号

**执行步骤：**
1. 调用 `mcp_patent-search__patsnap_fetch` 获取专利全文
2. 提取技术三要素（问题/方案/效果）
3. 拆解技术特征并分权重
4. 调用 `mcp_patent-search__patsnap_search` 检索申请人同领域已有专利，分析布局缺口
5. 调用 `mcp_web-search__web_search` 获取竞争对手最新专利动态
6. 生成**技术地图报告**，包含：
   - 当前保护边界分析
   - 六维创新挖掘（上位化/替换/组合拆分/场景延伸/改进/方法）
   - 优先级排序（P1/P2/P3）
   - 申请时机建议（是否需要加急/双报/PCT）

**输出：** `@session/patent-lifecycle/{日期}_{主题}_技术地图.html`

---

### 阶段2：FTO & 查新检索（V5深化：6路IPC×关键词系统覆盖）

**输入：** 技术方案描述 或 权利要求草稿

**V5执行步骤（必须完整执行，不得跳过）：**

1. **关键词矩阵构建**：中英文双语 × 同义词/上下位词扩展，生成3×3关键词矩阵
2. **IPC分类号确定**：主IPC + 副IPC（2～3个），逐组展开检索
3. **6路并行检索**（每路topk=50，共300条候选）：
   - 路线①：核心技术特征语义检索（semantic）
   - 路线②：关键词×主IPC定向检索（keyword+filter）
   - 路线③：关键词×副IPC定向检索（keyword+filter）
   - 路线④：申请人/竞争对手定向检索（filter）
   - 路线⑤：近似场景延伸检索（semantic）
   - 路线⑥：被引用/引用链追踪检索（keyword）
4. **去重合并**：同族折叠后精排，取Top20最相关文献
5. **特征比对**：逐条列出区别特征，确认新颖性结论
6. **覆盖率说明**：标注本次检索覆盖维度及局限性

**输出：** `@session/patent-lifecycle/{日期}_{主题}_查新报告.html`

---

### 阶段3b：技术交底书撰写（V5授权导向标准）

**输入：** 创新点描述 + 原专利号（可选）

**执行步骤：**
1. 调用 `mcp_patent-search__patsnap_fetch` 读取原专利全文作为背景参考
2. 按国家知识产权局标准格式撰写完整技术交底书，必须包含以下全部章节：
   - 一、所属技术领域
   - 二、现有技术背景及存在的问题（逐条列举缺陷，编号①②③，含对比文献专利号）
   - 三、发明目的（独立章节，明确解决的技术问题，与背景技术一一对应）
   - 四、技术方案（各模块详细说明，含参数，写入最宽实施方式段落）
   - 五、有益效果（表格形式：技术特征→解决的问题→有益效果→量化数据）
   - 六、附图说明（逐图列出图号+内容+全部附图标记对照表）
   - 七、具体实施例（至少3个，含真实参数+效果数据，含对比例1组）
   - 八、权利要求建议方向（含独立权利要求精准区别特征+方法权利要求+功能性权利要求）
   - 九、OA答复预案（预判2条审查意见+对应修改/争辩策略）
   - 十、IPC分类号建议（主+副）
   - 十一、发明人信息表
3. 执行**V5格式自检**（10项）：
   - 章节完整性核查（逐项对照标准格式）
   - 文本一致性核查（发明名称/技术方案/有益效果/附图标记四项一致性）
   - 实施例充分性评估（是否支撑权利要求保护宽度）
   - 独权区别特征明确性
   - 功能性备用权利要求存在性
   - 方法权利要求步骤完整性
   - 最宽实施方式段落存在性
   - 对比例数据存在性
   - OA预案完整性
   - IPC建议完整性
4. 输出**V5格式比对报告**（10项评分表 + 问题清单 + 修复建议）

**V5格式评分标准：**

| 评估项 | 满分 | 及格线 |
|--------|------|-------|
| 章节完整性（11章） | 100 | 95 |
| 技术描述深度 | 100 | 85 |
| 附图规范性 | 100 | 85 |
| 实施例充分性（含对比例） | 100 | 85 |
| 文本一致性 | 100 | 90 |
| 独权区别特征精准度 | 100 | 90 |
| OA预案完整性 | 100 | 85 |
| IPC建议完整性 | 100 | 90 |
| 功能性备用权利要求 | 100 | 85 |
| 最宽实施方式预埋 | 100 | 85 |

> **综合评分 ≥ 90分 = 可提交代理人撰写权利要求书的V5授权导向标准**
> 低于90分必须补全，不得跳过。

**输出：** 交底书正文（Markdown内嵌对话）+ V5格式比对报告

---

### 阶段3c：专利附图生成（V5：Base64内嵌，独立可分享）

**输入：** 附图说明文字 + 技术方案结构描述

**执行步骤：**
1. 调用 `runtime.apply_sync` 确保依赖已安装（ezdxf、matplotlib、numpy）
2. 调用 `files.write` 写入 Python 绘图脚本
3. 调用 `python.run` 执行绘图脚本，每件专利生成**4张PNG**：
   - 图1：整体结构示意图（含所有主要部件+标记编号+引线）
   - 图2：核心机构截面图/爆炸图（展示内部结构）
   - 图3：时序控制逻辑图/流程图（展示工作原理）
   - 图4：特性曲线/力学特性图（展示技术效果量化）
   - DXF：含标准图层的CAD结构图
4. 调用 `images.analyze` 核查附图质量（标记清晰度/结构准确性）
5. **V5新增：** 生成HTML时将所有PNG转为Base64内嵌，确保：
   - HTML文件可独立打开（无需依赖外部文件夹）
   - 附图可直接在浏览器中查看和缩放
   - 文件可单独发送给客户/代理人

**每件附图标准（4张PNG+1个DXF）：**

| 图号 | 内容 | 规格 |
|------|------|------|
| 图1 | 整体结构图 | 1200×900px，含标记编号+引线 |
| 图2 | 截面/爆炸图 | 1200×900px，展示内部结构 |
| 图3 | 时序/流程图 | 1200×700px，展示工作原理 |
| 图4 | 特性曲线图 | 1000×700px，量化技术效果 |
| DXF | CAD结构图 | 含7个标准图层 |

**DXF图层命名规范：**

| 图层名 | 内容 |
|--------|------|
| FRAME | 框架/壳体 |
| MECHANISM | 运动机构 |
| SENSOR | 传感器模块 |
| CONTROL | 控制模块 |
| ANNOTATION | 标注文字 |
| DIMENSION | 尺寸线 |
| CENTERLINE | 中心线 |

**输出路径：** `@session/patent_drawings/output_v2/`
**HTML内嵌：** 所有PNG以Base64格式嵌入最终HTML报告

### 阶段4：提交与审查应对（V5：OA预案已预埋，快速响应）

**输入：** 审查意见通知书文本

**执行步骤：**
1. 解析审查意见类型（新颖性/创造性/书面缺陷/单一性等）
2. 针对引用对比文件，调用 `mcp_patent-search__patsnap_fetch` 获取全文分析
3. 对照阶段3b中已预埋的OA预案，快速匹配对应策略
4. 生成**OA答复策略建议**：
   - 修改方案（可修改范围边界分析）
   - 争辩要点（与对比文件的区别特征及技术效果）
   - 修改 vs 争辩 vs 组合策略推荐
   - 驳回后救济路径（复审/行政诉讼）

**输出：** `@session/patent-lifecycle/{日期}_{专利号}_OA答复建议.md`

## 六、创新点挖掘专项流程（从授权专利挖后续案）

### Step 1：精读原专利
- `mcp_patent-search__patsnap_fetch` 获取完整权利要求+说明书
- 提取技术三要素
- 拆解所有技术特征

### Step 2：六维挖掘

| 维度 | 挖掘方向 | 输出候选案 |
|------|---------|---------  |
| 上位化 | 抽象为更宽的技术概念 | 1-2件 |
| 替换替代 | 功能相同结构不同的替代方案 | 2-3件 |
| 组合拆分 | 子组合单独成立的方案 | 1-2件 |
| 场景延伸 | 同原理应用于新场景 | 2-3件 |
| 改进优化 | 解决原专利遗留问题 | 1-2件 |
| 制备方法 | 产品→方法专利补充 | 1件 |

### Step 3：候选案深化查新验证（V5：6路检索）
对每个候选方案执行6路并行检索（同阶段2），每路topk=50，输出区别特征确认表。

### Step 4：优先级排序输出
按**商业价值 × 新颖性空间 × 申请紧迫性**三维评分，给出 P1🔴/P2🟠/P3🟡 排序

### Step 5：对P1方向启动V5授权导向交底书（阶段3b）+ 附图生成（阶段3c）

**V5新增：近似现有技术风险预警**
- 对每件候选案，重点标注"⚠️潜在近似现有技术"（如来自其他领域但功能相近的方案）
- 给出针对性权利要求撰写策略，预防审查员组合引用

**输出：** `@session/patent-lifecycle/{日期}_{原专利号}_后续挖掘报告_v5.html`

## 八、质量控制红线（V5：12条）

❌ 禁止：给出确定性的侵权结论（只能给倾向性分析，最终结论需律师确认）
❌ 禁止：编造专利法条款或审查指南规则
❌ 禁止：将查新结论等同于审查员结论
❌ 禁止：超出原始申请范围建议修改权利要求
❌ 禁止：技术交底书综合评分低于90分即提交
❌ 禁止：附图使用相对路径引用外部文件（必须Base64内嵌）
❌ 禁止：跳过V5授权导向9项强制检查（GRANT_MODE=on时）
❌ 禁止：说"开始"后跳过附图生成步骤（必须完整执行阶段3c）

✅ 必须：每项核心结论标注[S#]来源
✅ 必须：区分"事实陈述"与"专业判断"
✅ 必须：重要法律节点提醒附上官方依据
✅ 必须：报告末尾标注"本报告仅供参考，不构成法律意见"

## 十三、注意事项

1. **法律边界**：所有输出均为辅助参考，不构成正式法律意见，重大决策须由执业专利代理师/律师确认
2. **时效敏感**：专利法律期限（优先权期、年费期、答复期）误期后果严重，提醒用户务必以官方通知为准
3. **保密原则**：技术交底书内容属于高度机密，不应在非必要环节扩散
4. **中国法规优先**：默认适用中国专利法及专利审查指南，海外申请另行说明适用法规
5. **数据来源透明**：所有检索结论标注来源，不确定内容明确标注"待核实"
6. **附图说明**：生成的PNG/DXF为工程示意图，提交国知局前须委托专业制图人员按官方规范精修
7. **交底书评分**：综合评分须≥90分方可提交代理人，低于此分值须先补全缺失内容
8. **流程完整性**：V5起，凡用户说"开始"/"执行"/"立即执行"，必须完整走完附图生成步骤，不得以"历史结果"替代当前执行
