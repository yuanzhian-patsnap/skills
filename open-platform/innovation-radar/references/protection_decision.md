# 保护形式决策树

本文件定义三种保护形式的判断逻辑：申请专利、商业秘密、暂不保护。
供 SKILL.md 主流程在完成新颖性快筛后使用。

---

## 核心决策流程

```
技术点确认可保护
        ↓
问题1：技术方案能被逆向工程获取吗？
   ├─ 能（产品结构、化学配方、外观）→ 优先申请专利
   └─ 不能（内部算法、工艺参数、训练数据）→ 进入问题2

问题2：是否计划对外许可、融资展示、或参与标准制定？
   ├─ 是 → 必须申请专利（商业秘密不可交易、不可主张）
   └─ 否 → 进入问题3

问题3：竞争对手是否可能独立研发出相同方案？
   ├─ 可能（通用技术路线）→ 抢先申请专利
   └─ 不太可能（高度依赖内部数据/硬件）→ 商业秘密性价比更高

问题4：技术生命周期是否超过20年？
   ├─ 是（基础技术、平台技术）→ 考虑商业秘密（专利有期限）
   └─ 否（迭代快的领域）→ 专利保护窗口足够
```

---

## 三种保护形式对比

| 维度 | 申请专利 | 商业秘密 | 暂不保护 |
|---|---|---|---|
| **保护期** | 发明20年/实用新型10年 | 永久（只要保密） | 无 |
| **可交易性** | ✅ 可许可、转让、质押 | ❌ 难以对外交易 | 无 |
| **公开要求** | 必须公开技术方案 | 必须严格保密 | 无要求 |
| **维权方式** | 专利侵权诉讼 | 不正当竞争诉讼 | 无法维权 |
| **成本** | 申请费+维持年费 | 保密管理成本 | 零 |
| **适用场景** | 可逆向/对外许可/竞争激烈 | 不可逆向/内部长期使用 | 现有技术/业务逻辑 |

## 各场景下的保护建议模板

### 建议申请专利时的输出格式
```
保护建议：申请专利（[发明专利/实用新型/外观设计]）
专利类型建议：[方法权利要求 / 装置权利要求 / 用途权利要求]
理由：[一句话，说明为什么专利比商业秘密更合适]
紧急程度：[高：7天内提交交底书 / 中：1个月内 / 低：季度内]
```

### 建议商业秘密时的输出格式
```
保护建议：商业秘密
保密措施建议：限制知情人范围、签署NDA、技术文档加密存储
理由：[一句话，说明为什么商业秘密比专利更合适]
注意：如后续涉及对外合作，需重新评估是否转为专利保护
```

### 建议暂不保护时的输出格式
```
保护建议：暂不保护
理由：[从以下选一个] 
  - 属于常规技术手段，无新颖性
  - 现有技术中已有大量相似方案
  - 属于纯业务逻辑，不构成技术方案
  - 技术特征描述不完整，待补充后重新评估
```

## 特殊场景处理

### 软件/算法类技术
- 纯算法逻辑本身不可专利，但**结合具体技术效果的实现方案**可以
- 判断标准：是否解决了技术问题、产生了技术效果
- 建议：描述时强调"在XX硬件/系统上的具体实现"而非算法本身

### AI/机器学习类技术
- 模型结构、训练方法、数据处理流程均可申请
- 关键：强调技术改进（速度/精度/资源占用），而非商业效果
- 商业秘密选项：训练数据集、超参数配置往往更适合商业秘密

### 硬件+软件协同方案
- 优先申请系统级专利（覆盖硬件+软件整体）
- 再分别申请方法专利和装置专利，形成专利组合
