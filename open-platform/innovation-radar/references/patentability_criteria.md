# 可专利性判断标准

本文件定义新颖性潜力和技术完整度两个评估维度的打分标准。
这是"研发创新点雷达"的核心价值评分体系。

---

## 价值评分体系：两个独立维度

### 维度一：新颖性潜力（Novelty Potential）

**定义：** 该技术点在全球专利数据库中尚未被公开的可能性。
**评分：** 高 / 中 / 低

| 等级 | 判断标准 | MCP检索结果参考 |
|---|---|---|
| **高** | 技术手段描述独特，检索未见高度相似方案；或技术组合方式新颖 | 语义检索相似度最高结果 <70%；关键词检索命中量 <5件 |
| **中** | 存在类似方向的研究，但本方案的具体实现有明显差异 | 相似度 70-85%；命中量 5-20件，但权利要求有区别 |
| **低** | 已有大量现有技术，本方案属于常规改进或已公开方案 | 相似度 >85%；命中量 >20件，核心特征高度重合 |

**无MCP检索时的纯文本判断规则：**
- 技术手段包含具体数值/参数范围 → 倾向"中"（参数本身可能是新颖点）
- 技术手段使用业界通用术语描述 → 倾向"低"（先标低，等检索确认）
- 技术手段描述了罕见的组合方式 → 倾向"中高"
- 发明人明确表示"业界从未这样做过" → 倾向"高"（但需验证）

---

### 维度二：技术完整度（Technical Completeness）

**定义：** 技术三要素（问题-手段-效果）的完整程度，决定能否直接推进专利申请。
**评分：** 完整 / 需补充 / 碎片

| 等级 | 判断标准 | 对应行动 |
|---|---|---|
| **完整** | 三要素均清晰，手段可构成独立权利要求雏形，效果有数据支撑 | 直接建议启动技术交底书撰写 |
| **需补充** | 三要素有1-2个模糊或缺失，但核心技术思路清晰 | 输出追问问题清单，补充后可推进 |
| **碎片** | 仅有模糊的技术方向，三要素均不完整，无法构成权利要求 | 标记"待确认"，建议与发明人面谈 |

---

## 两维度组合评估矩阵

```
           技术完整度
           完整      需补充     碎片
新    高  │ 🔴立即行动 │ 🟡优先补充 │ 🟡优先补充 │
颖        │           │           │           │
性    中  │ 🟡建议推进 │ 🟡建议推进 │ ⚪待确认   │
潜        │           │           │           │
力    低  │ ⚪暂不保护 │ ⚪暂不保护 │ ⚪排除     │
```

### 各象限解读

**🔴 立即行动（新颖性高 × 完整）**
> 最有价值的创新点。建议7天内提交技术交底书，同时确认公开风险。

**🟡 优先补充（新颖性高/中 × 需补充/碎片）**
> 有保护价值，但信息不足。输出追问问题清单，推动发明人尽快补充。

**🟡 建议推进（新颖性中 × 完整）**
> 有一定保护价值。建议先做深度查新，确认与现有技术的具体区别后再推进。

**⚪ 待确认（新颖性中/低 × 碎片）**
> 信息太碎片，无法判断价值。标记"待确认"，等待发明人补充后重新评估。

**⚪ 暂不保护（新颖性低 × 任何完整度）**
> 现有技术已有大量相似方案。说明原因，建议归档，不消耗IP资源。

---

## 评分输出格式

```
新颖性潜力：🔴 高 / 🟡 中 / ⚪ 低
技术完整度：完整 / 需补充 / 碎片
综合评级：🔴 立即行动 / 🟡 建议推进 / ⚪ 待确认 / ⚪ 暂不保护

评分依据（新颖性）：[说明检索结果或文本判断依据，1-2句]
评分依据（完整度）：[说明缺失的是哪个要素，或为何认为完整]
```

---

## 注意事项

- **新颖性潜力"高"不等于一定能授权**：创造性（非显而易见性）需要更深度的分析，本 Skill 只做初筛
- **技术完整度"完整"不等于权利要求已经写好**：需要专业代理师进一步撰写和优化
- **两个维度均独立打分，不合并为单一分数**：避免黑箱，让发明人和IP工程师能理解判断依据
