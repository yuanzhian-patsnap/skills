# 追问问题库

本文件定义按创新类型分类的追问问题，用于在技术三要素不完整时生成"待补充清单"。
追问目标：帮发明人补全构成权利要求所需的最小必要信息。
输出方式：文字清单，用户可直接转发给发明人填写。

---

## 追问触发逻辑

```
技术三要素提取结果
├─ 三要素完整 → 直接进入MCP检索，不追问
├─ 缺"技术手段" → 触发 [手段类追问]
├─ 缺"技术效果" → 触发 [效果类追问]
├─ 缺"技术问题" → 触发 [问题类追问]
└─ 三要素均不完整 → 触发 [通用基础追问] + 建议重新提供完整描述
```

---

## 通用基础追问（三要素均模糊时使用）

```
📋 待补充信息清单 — 请发明人确认以下问题：

1. 【技术问题】这个方案要解决的核心技术问题是什么？
   现有方案（或之前的方案）为什么解决不好这个问题？

2. 【技术手段】具体用了什么新的技术方法、结构或材料？
   与原来的做法相比，"新"在哪里？（哪怕只改了一个参数或步骤）

3. 【技术效果】改进后，实际达到了什么可量化的效果？
   有没有测试数据或对比数据支撑？

4. 【公开风险】这个方案是否已经在以下场合提及或使用：
   □ 发表论文 □ 学术会议演讲 □ 开源代码 □ 公开演示
   □ 与外部合作方讨论 □ 产品已上市销售
```

## 按创新类型的专项追问

### 新方法类追问

```
📋 待补充信息清单 — 方法类技术点：

1. 这个方法包含哪几个关键步骤？步骤之间有没有严格的先后顺序？
2. 每个步骤的执行条件是什么？（触发条件、输入参数、约束范围）
3. 与现有方法相比，关键步骤的差异在哪里？
4. 这个方法是否依赖特定硬件？如果换硬件，方法还有效吗？
5. 效果数据：在什么测试条件下，得到了什么具体数值？
```

### 新结构 / 新装置类追问

```
📋 待补充信息清单 — 结构/装置类技术点：

1. 新增或改变的是哪个部件/模块？与其他部件如何连接？
2. 这个结构改变解决了什么具体问题？（原结构有什么缺陷？）
3. 是否有示意图、CAD图或实物照片？（有助于后续撰写说明书）
4. 这个结构是否已经有实物样机或原型验证？
5. 与现有同类产品的结构对比，本质区别在哪个关键部件上？
```

### 新参数 / 新配方类追问

```
📋 待补充信息清单 — 参数/配方类技术点：

1. 关键参数的具体数值范围是什么？（上下限各是多少？）
2. 这个参数范围是如何确定的？有没有对比实验？
3. 在这个参数范围之外，效果会发生什么变化？
4. 该参数改变带来的技术效果是否"出乎意料"？（与预期不同才更有价值）
5. 实验重复性如何？不同批次/环境下效果是否稳定？
```

### 新效果 / 新用途类追问

```
📋 待补充信息清单 — 新用途类技术点：

1. 原技术/方法最初是为什么场景设计的？
2. 发现它在新场景中有效是在什么情况下？（偶然发现还是系统验证？）
3. 在新场景下的具体效果数据是什么？
4. 新场景下是否需要对原技术做任何适应性修改？
5. 是否有文献或竞对产品在新场景下用过类似技术？
```

### 新流程 / 新系统类追问

```
📋 待补充信息清单 — 流程/系统类技术点：

1. 新流程与旧流程的核心差异在哪几个节点上？
2. 各模块/步骤之间的数据流/控制流是什么？
3. 整体系统产生的效果，是否是各部分单独无法实现的？（组合创新的核心）
4. 系统中哪个模块是自研的，哪些是调用现有技术/开源库？
5. 如果只保护一个最核心的子系统，应该是哪个？
```

## 公开风险专项追问（所有类型均需确认）

```
📋 公开风险确认清单（所有技术点必查）：

□ 是否已投稿或发表学术论文？（投稿即视为公开风险）
□ 是否在学术会议、技术论坛、行业展会做过演讲或展示？
□ 是否有相关代码已上传至 GitHub 或其他开源平台？
□ 是否在公司外部的技术博客、公众号、视频中提及？
□ 是否在与客户、供应商、投资人的商务谈判中详细描述过？
□ 基于该技术的产品是否已对外销售或公开发布？

⚠️ 若以上任何一项为"是"，请告知具体时间——
   中国专利法设有6个月的优惠期（部分情形），需尽快评估。
```
