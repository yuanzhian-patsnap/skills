# 产品/技术领域立项全景分析技能 v3.0

## 一、技能基础信息

### 技能名称
产品/技术领域立项全景分析（国内+海外·龙头强制核查·技术热点/空白·全球受理局布局·HTML报告）

### 技能目标
1. 同时检索**国内+海外**企业的全球专利数据，不排除中国申请人
2. 强制核查行业公认龙头企业（基于预置名单+用户补充），确保龙头不遗漏
3. **新增**：技术热点识别（高密度方向）+ 技术空白点挖掘（低密度/待探索方向）
4. **新增**：申请人全球目标市场分析——受理局维度布局热力图与趋势
5. 结合市场规模、技术格局、竞争梯队、机会与风险，输出完整立项分析报告
6. 输出**优化版 HTML 报告**（专业配色、模块化卡片、可视化图表区）

### 触发条件
输入格式：产品/技术领域名称（+ 可选：核心关键词 + 已知龙头企业名单）
示例输入1：油烟机
示例输入2：新能源汽车热管理；thermal management；比亚迪,宁德时代,法雷奥
示例输入3：工业机器人减速器；harmonic drive,RV reducer；纳博特斯克,住友,哈默纳科

## 三、执行步骤详解

### 步骤1：输入解析
- 从用户输入提取：产品/领域名称、核心关键词（中英文）、用户补充的龙头企业
- 自动生成领域同义词、英文译名、IPC/CPC 分类号候选列表
- 输出：标准化检索要素

### 步骤2：龙头企业名单预置（⚠️ 核心机制，防遗漏）
- **规则**：在执行任何检索之前，先确定该领域全球及中国市场的 Top5-10 龙头企业名单，明确写出企业全称及常用简称
- **强制核查**：无论检索结果是否包含这些企业，都必须对每家龙头企业单独执行一次专利检索
- **中国龙头不可省略**：国内龙头与海外龙头同等重要，必须全部纳入分析
- 输出示例格式：
  ```
  预置龙头名单（油烟机领域）：
  国内：方太（FOTILE）、老板电器（Robam）、海尔（Haier）、美的（Midea）、华帝（VATTI）
  海外：BSH（博西）、Electrolux（伊莱克斯）、Miele（美诺）、Broan-NuTone、Faber
  ```

### 步骤3：市场规模与趋势检索
- 通过网络检索获取该领域近3年市场规模、增速、主要驱动因素、政策背景
- 记录数据来源，标注口径差异
- 输出：市场规模表格 + 增长驱动因素清单

### 步骤4：专利全局检索（国内+海外，≥50条）
- 检索范围：全球专利，**包含中国申请人**，涵盖 CN/US/EP/JP/KR/WO 等主要局
- topk 建议 50，若结果 < 50 自动扩展关键词或放宽时间范围至 2015 年
- **⚠️ 不得排除中国申请人**

### 步骤5：龙头企业强制补查（⚠️ 防遗漏核心步骤）
- 对步骤2预置的每家龙头企业，**单独执行一次** assignee 过滤检索
- 逐一记录每家龙头的专利数量、核心技术方向、代表专利（至少1件）
- 输出：龙头企业补查汇总表（企业名 | 检索到的专利数 | 核心技术方向 | 代表专利）

### 步骤6：技术热点与空白点分析（⚠️ v3.0 新增）

#### 6.1 技术热点识别
- **定义**：近3年专利申请量显著增加、多家龙头同时布局的技术方向
- **方法**：
  - 统计各技术方向专利数量占比，占比 > 15% 或近3年增速 > 50% 的方向标记为热点
  - 交叉验证：同一方向有3家以上主要申请人布局，视为竞争热点
  - 可结合 IPC 小类聚类分析
- **输出格式**：
  ```
  🔥 热点方向1：智能控制与传感（占比约30%，2022-2025年增速+80%）
    代表申请人：方太、海尔、美的
    代表专利：CN114322022A、CN116700371A
    热点判断依据：多家龙头密集布局，与IoT/AI趋势深度绑定
  ```

#### 6.2 技术空白点挖掘
- **定义**：技术上可行、市场有需求、但现有专利布局稀疏的方向
- **识别方法**（优先级排序）：
  1. **低密度方向**：全部检索结果中某技术分支 < 3% 占比，且近3年无新申请
  2. **交叉融合空白**：两个成熟技术方向的交叉点（如"自清洁 × AI视觉监测"）尚无专利
  3. **用户痛点映射**：行业公认的用户痛点（通过市场检索获取），对应技术方向布局稀少
  4. **龙头回避区**：主要龙头均未布局的细分场景（如商用/无管道/车载场景）
- **输出格式**：
  ```
  ⬜ 空白点1：车载/移动场景油烟净化
    空白判断依据：检索样本中无相关专利，市场有移动餐饮场景需求
    潜在机会评级：★★★（中等）
    建议切入路径：小型静电净化模块 + 车规级认证
  ```
- **每个技术方向分析完后，明确标注"热点 🔥 / 成熟 🟡 / 空白 ⬜ / 衰退 🔻"**

### 步骤7：申请人全球目标市场分析——受理局布局（⚠️ v3.0 新增）

#### 7.1 分析目标
揭示主要申请人在全球各受理局的专利布局密度，判断其全球市场战略意图。

#### 7.2 分析维度
- **受理局覆盖广度**：申请人在 CN / US / EP / JP / KR / WO / DE / AU / CA 等主要局的布局情况
- **PCT 利用率**：是否通过 PCT（WO）进行国际申请，反映国际化意愿
- **重点布局局**：申请量最多的3个受理局，反映核心目标市场
- **空白局**：主要竞争对手已布局但本申请人未布局的受理局，揭示防御空白

#### 7.3 数据获取方法
- 对每家龙头企业，在步骤5的补查检索中记录专利的 jurisdiction 字段分布
- 或对 jurisdiction 维度逐一用 filter 检索：`assignees: ["方太"] + jurisdiction: ["US"]`
- 汇总为受理局分布矩阵：行=申请人，列=受理局（CN/US/EP/JP/KR/WO/其他）

#### 7.4 输出格式
```
申请人全球受理局布局矩阵（示例）：
| 申请人 | CN | US | EP | JP | KR | WO | 布局广度 | PCT利用率 |
|--------|----|----|----|----|----|----|---------|---------|
| 方太   | ●● | ●  | ●  | ○  | ○  | ●  | 中等     | 有      |
| 海尔   | ●● | ●● | ●  | ●  | ●  | ●● | 广泛     | 高      |
| BSH    | ●  | ●● | ●● | ●  | ○  | ●● | 广泛     | 高      |
（● = 有布局  ●● = 重点布局  ○ = 无/极少）

关键发现：
- 国内龙头（方太/老板）以 CN 为核心，US/EP 布局有限，存在海外防御空白
- 海外品牌（BSH/Electrolux）在 EP/US 形成包围，CN 反向布局值得关注
- PCT 路径：方太/海尔均有 WO 申请，具备进攻海外意愿
```

#### 7.5 市场战略解读
- **进攻型**：在多个海外局（US/EP/JP）均有布局，说明有出海意图
- **防御型**：仅 CN 大量布局，海外几乎空白，说明以本土防御为主
- **平衡型**：CN 为主，WO/US 适度布局，兼顾本土与国际
- 给出每家龙头的市场战略类型判断

### 步骤8：竞争梯队分析
- 按专利数量、技术深度、市场地位，将竞争者分为3个梯队
- 每个梯队列出企业名、核心优势、布局重点、全球市场战略类型（结合步骤7）

### 步骤9：机会与风险评估
- 机会维度：市场空白、**技术空白（来自步骤6）**、政策红利、新兴场景
- 风险维度：头部垄断、价格战、**专利壁垒（结合热点方向）**、行业周期
- 各给出3-5条，标注信号强度（高/中/低）

### 步骤10：差异化立项建议
- 基于**技术空白点（步骤6）+ 受理局空白（步骤7）+ 市场机会（步骤9）**，提出2-4个可行立项方向
- 每个方向包含：核心价值、技术路径、差异化点、建议优先申请的受理局、潜在风险

### 步骤11：生成优化版 HTML 报告（⚠️ v3.0 优化）

#### 11.1 整体设计规范
- **配色方案**：主色 `#1a56db`（蓝）+ 辅色 `#0e9f6e`（绿）+ 警示 `#e02424`（红）+ 背景 `#f9fafb`
- **字体**：`font-family: 'Segoe UI', 'PingFang SC', 'Microsoft YaHei', sans-serif`
- **布局**：最大宽度 `1200px`，左侧固定导航栏（桌面端），内容区卡片化
- **响应式**：支持移动端折叠导航

#### 11.2 必须包含的模块（按顺序）

```
① 封面/标题区
   - 报告标题、技术领域、生成日期、版本号
   - 蓝色渐变背景（#1a56db → #1e429f）+ 白色文字
   - 包含一行「执行摘要」（3句话概括核心结论）

② 导航栏（左侧固定，桌面端）
   - 锚点导航，含章节编号和标题
   - 当前章节高亮显示

③ 市场概览（Section 1）
   - 市场规模数字大卡片（带增速箭头图标）
   - 驱动因素水平卡片列表（emoji 图标）

④ 技术格局（Section 2）
   - 各技术方向水平进度条（反映专利占比）
   - 每个方向标注状态徽章：🔥热点 / 🟡成熟 / ⬜空白 / 🔻衰退

⑤ 技术热点与空白点（Section 3）  ← v3.0 新增
   - 热点卡片（红色左边框）+ 空白点卡片（灰色左边框，虚线）
   - 空白点卡片标注「机会评级 ★★★」和「建议切入路径」

⑥ 全球受理局布局（Section 4）  ← v3.0 新增
   - HTML 表格形式的受理局矩阵
   - 用色块深浅表示布局强度（深蓝=重点，浅蓝=有布局，灰=无布局）
   - 下方附「市场战略类型」标签（进攻型/防御型/平衡型）

⑦ 竞争格局（Section 5）
   - 三梯队色块区分（第一梯队深蓝，第二梯队蓝绿，第三梯队浅灰）
   - 每家企业卡片含：企业名、核心优势标签、代表专利编号、全球战略类型

⑧ 机会与风险（Section 6）
   - 左右双栏：绿色机会列表 vs 红色风险列表
   - 每条含信号强度标签（高/中/低）

⑨ 立项建议（Section 7）
   - A/B/C 编号卡片，蓝色渐变标题
   - 每张卡片含：核心价值、技术路径、差异化点、建议申请局、潜在风险

⑩ 综合评分（Section 8）
   - 五维雷达评分（市场空间/技术可行性/竞争强度/政策友好度/专利布局难度）
   - 表格形式，带星级 ⭐ 可视化

⑪ 参考文献（Section 9）
   - 专利证据（含智慧芽链接）和网络来源分类列示
   - 标注 [S编号] 来源标识
```

#### 11.3 HTML 技术规范
- 完整自包含（无外部 CDN 依赖，CSS/JS 全部内嵌）
- 导航锚点使用 `id` 属性，JS 实现平滑滚动和当前章节高亮
- 进度条用纯 CSS 动画实现（页面加载时渐入）
- 受理局矩阵用 `<table>` 实现，单元格背景色用内联 style 控制
- 所有外部链接 `target="_blank" rel="noopener"`
- 文件编码 `UTF-8`，`<meta charset="UTF-8">`

#### 11.4 报告输出路径
- 默认保存至：`@session/reports/[领域名称]_project_analysis_v3.html`
- 文件名使用领域关键词，避免中文文件名（使用拼音或英文）

## 四、质量保障规则

| 规则编号 | 规则内容 |
|---------|---------|
| QA-01 | **国内龙头不可省略**：凡中国市场份额 > 10% 的企业，必须出现在报告竞争格局章节中 |
| QA-02 | **龙头补查强制执行**：即使全局检索中某龙头结果很少，也必须单独补查并说明原因 |
| QA-03 | **来源可追溯**：市场数据标注来源网址，专利数据标注检索条件和返回数量 |
| QA-04 | **数据口径说明**：若不同来源数据存在出入，必须注明口径差异，不得取平均值代替 |
| QA-05 | **样本范围说明**：专利统计结论必须标注"基于topk检索样本"，非完整组合分析 |
| QA-06 | **遗漏自检**：报告完成后，对照预置龙头名单逐一确认每家企业是否已体现 |
| QA-07 | **热点/空白标注完整**：每个技术方向必须标注状态（🔥热点/🟡成熟/⬜空白/🔻衰退），不得留空 |
| QA-08 | **受理局矩阵必须输出**：凡有2家以上申请人数据，必须生成受理局矩阵表，不得以文字替代 |
| QA-09 | **HTML 报告自包含**：不得引用外部 CDN，所有 CSS/JS 必须内嵌，确保离线可用 |

## 五、检索示例

### 全局检索（含国内外所有申请人）
```json
{
  "search_strategy": ["keyword", "filter"],
  "keywords": ["油烟机", "抽油烟机", "range hood", "kitchen hood"],
  "filters": { "ipc": ["F24C15/20"], "date_from": 20180101 },
  "sources": ["patent"],
  "topk": 50
}
```

### 龙头企业专项补查（方太）+ 受理局分析
```json
{
  "search_strategy": ["keyword", "filter"],
  "keywords": ["油烟机", "抽油烟机", "range hood"],
  "filters": { "assignees": ["方太", "FOTILE", "浙江方太"] },
  "sources": ["patent"],
  "topk": 30
}
```

### 受理局单点验证（方太 × US）
```json
{
  "search_strategy": ["filter"],
  "filters": {
    "assignees": ["方太", "FOTILE"],
    "jurisdiction": ["US"]
  },
  "sources": ["patent"],
  "topk": 10
}
```
