# 央企外部科技成果吸纳工作流

## 背景与定位

本技能面向**央企科技管理部门负责人**，解决外部科技成果吸纳工作中的六大核心难点：

1. 技术评估能力不足
2. 知识产权边界不清
3. 央企体制与市场定价张力
4. "最后一公里"内部承接问题
5. 考核激励与风险容忍矛盾
6. 信息不对称与中介市场失灵

通过调用智慧芽MCP能力，将原本依赖专家经验和人工检索的工作转化为**可复用、可追溯、可审计**的标准化流程。

---

## 触发条件

用户输入以下任意内容时启动本技能：

- 提及"外部科技成果"、"技术引进"、"成果转化"、"吸纳外部技术"
- 提及"扫描技术领域"、"专利情报"、"成果情报简报"
- 提供专利号清单并要求评级或尽职调查
- 描述一项拟引进技术并询问转化路径
- 明确要求"立项预审"、"决策证据链"、"引进模式"

## 五大工作模式

### 模式 A：情报扫描模式

**适用场景**：科技管理部门定期主动扫描某技术领域，发现潜在引进标的。

**输入**：
- 技术领域关键词（必填）
- 重点关注的应用场景或技术方向（可选）
- 扫描时间范围（可选，默认近3年）

**执行步骤**：

```
Step 1 - 专利扫描
调用 patent.search（mcp_patent-search__patsnap_search）
  参数：search_strategy=["semantic","keyword"]
       semantic_query=技术领域自然语言描述
       keywords=提取的核心技术关键词（3-6个）
       sources=["patent"]
       topk=30
       filters.date_from=近3年起始日期

Step 2 - 论文扫描
调用 paper.search（mcp_patent-search__patsnap_search）
  参数：search_strategy=["semantic","keyword"]
       semantic_query=同上
       keywords=同上
       sources=["paper"]
       topk=20

Step 3 - 结果整理
按以下维度对检索结果分类：
  - 高校/科研院所成果（潜在许可/转让标的）
  - 初创企业成果（潜在合作开发/入股标的）
  - 国外机构成果（潜在引进/合作标的）
  - 高被引论文（尚未专利化的前沿研究，主动对接价值）

Step 4 - 输出成果
生成《月度外部技术成果扫描简报》，包含：
  - 技术领域热点摘要（近期专利/论文趋势）
  - 重点标的清单（Top10，含机构、专利号/论文DOI、简要评价）
  - 建议优先跟进标的（3-5项）
  - 建议下一步动作（进入模式C评级 或 直接联系）
```

**输出物**：Markdown格式扫描简报

### 模式 B：供给方评估模式

**适用场景**：已锁定一家机构/企业，需评估其整体技术实力和合作价值。

**输入**：
- 供给方机构/企业名称（必填）
- 关注的具体技术方向（可选）

**执行步骤**：

```
Step 1 - 公司技术画像
加载并调用 company-tech-profile 技能
  输入：公司名称 + 技术方向

Step 2 - 竞争格局对比
加载并调用 competitive-landscape 技能
  目的：了解该供给方在行业中的技术地位，
        识别同类技术的其他供给方（备选方案）

Step 3 - 综合评价
输出供给方评估报告，包含：
  - 技术实力概览（专利数量/质量/技术方向）
  - 行业地位（与竞争者的相对位置）
  - 合作价值判断（技术互补性、战略协同性）
  - 替代供给方清单（为谈判提供备选支撑）
  - 建议合作模式
```

**输出物**：供给方评估报告（Markdown）

---

### 模式 C：资产分级模式

**适用场景**：供给方提供专利号清单，需批量完成技术与法律价值评估。

**输入**：
- 专利号清单（必填，支持CN/US/EP/WO等）
- 企业重点关注的技术方向（可选，用于匹配度评分）

**执行步骤**：

```
Step 1 - 批量专利数据获取
调用 patent.fetch（mcp_patent-search__patsnap_fetch）
  参数：keys=专利号列表
       key_type="pn"
       module=["basic","legal","family","citation"]
  说明：获取每件专利的全量数据（基本信息+法律状态+家族+引用）

Step 2 - 资产分级评审
加载并调用 patent-asset-grading 技能
  评分维度（五维）：
    - 技术关联性：与企业研发方向的匹配度
    - 权利要求强度：独立权利要求的保护范围扎实程度
    - 市场覆盖度：主要目标市场的专利布局
    - 剩余保护期：结合法律状态的可用年限
    - 被引用/交叉价值：高被引代表核心技术节点

Step 3 - IP边界核查
基于Step 1获取的 legal 和 family 数据，逐件核查：
  □ 是否存在质押记录
  □ 是否存在许可在先（独占许可/普通许可）
  □ 是否存在权属争议或共有情况
  □ 海外同族专利的覆盖国别是否与企业市场匹配
  □ 引用链中是否存在上游基础专利依赖风险

Step 4 - 输出成果
生成《专利资产分级评审报告》，包含：
  - 资产分级汇总表（S/A/B/C/D五档）
  - 各维度评分明细
  - IP边界核查结论（红/黄/绿三色标注）
  - 重点关注清单（S/A级 + 有IP风险的）
  - 谈判优先级建议
```

**输出物**：专利资产分级评审报告（Markdown + 可选Word）

### 模式 D：转化路径模式

**适用场景**：已确定拟引进的技术/专利，需确定最优引进模式和对接目标。

**输入**：
- 拟引进专利号或技术描述（必填）
- 企业自身技术现状（可选）
- 预算范围和时间窗口（可选）

**执行步骤**：

```
Step 1 - 转化路径发现
加载并调用 tech-transfer-target-discovery 技能
  输入：专利号或技术描述
  自动识别：许可 / 转让 / 合作开发 / 作价入股 四种路径的适用性
  输出：目标企业/机构短名单 + 推荐对接路径

Step 2 - 路径适配评估
对每种路径进行可行性评估：

  许可模式：
    适用条件：技术已成熟、企业有能力独立实施、对方愿意保留所有权
    国资合规要点：需评估许可费标准是否在合理范围内
    风险点：独占许可条款谈判、许可范围边界

  转让模式：
    适用条件：技术关键性高、对方愿意完全退出、一次性定价可接受
    国资合规要点：需经过国资评估备案，防止国有资产流失认定
    风险点：转让价格合规性、瑕疵担保条款

  合作开发模式：
    适用条件：技术尚未成熟（TRL 4-6级）、双方均有研发能力
    国资合规要点：约定合作成果归属比例，避免后续争议
    风险点：成果归属条款、保密义务、退出机制

  作价入股模式：
    适用条件：对方为初创企业、技术具有高成长潜力
    国资合规要点：需通过资产评估机构出具评估报告，国资委备案
    风险点：估值合规性、股权稀释、退出通道

Step 3 - 输出推荐方案
生成《技术引进路径推荐报告》，包含：
  - 推荐路径（第一优选 + 备选）及理由
  - 关键谈判要点清单
  - 国资合规操作要点
  - 内部承接单元建议（哪个业务单元最适合）
  - 下一步行动计划（含时间节点建议）
```

**输出物**：技术引进路径推荐报告（Markdown）

---

### 模式 E：立项决策模式

**适用场景**：需要生成完整的尽职调查证据链，支撑立项审批和尽职免责。

**输入**：
- 已完成模式C（资产分级）和模式D（转化路径）的输出结果
- 企业内部立项模板（可选上传）

**执行步骤**：

```
Step 1 - 研发立项预审
加载并调用 rd-initiation-review 技能
  目的：从技术创新性、市场可行性、战略契合度三个维度
        完成独立的项目预审评价

Step 2 - 证据链汇总
整合以下材料形成完整证据包：
  □ 情报扫描简报（模式A输出，证明"市场调研已完成"）
  □ 供给方评估报告（模式B输出，证明"供给方尽调已完成"）
  □ 专利资产分级报告（模式C输出，证明"IP评估已完成"）
  □ 技术引进路径报告（模式D输出，证明"方案比选已完成"）
  □ 立项预审报告（Step 1输出，证明"独立评审已完成"）

Step 3 - 尽职免责声明生成
生成标准化《技术引进尽职调查说明》，包含：
  - 调查范围与方法说明
  - 主要发现摘要
  - 风险提示及应对措施
  - 决策依据说明
  - 免责边界声明

Step 4 - 输出决策包
生成《技术引进立项决策包》，包含：
  - 项目概况一页纸（供管理层快速决策）
  - 完整证据链目录（供审计/合规查阅）
  - 建议决策结论（通过/附条件通过/暂缓/否决）
```

**输出物**：立项决策包（Markdown，可转Word）

## 完整工作流（五模式串联）

```
战略需求输入
     │
     ▼
[模式A] 情报扫描
patent.search + paper.search
→ 月度扫描简报 + 重点标的清单
     │
     ▼（选定标的）
[模式B] 供给方评估
company-tech-profile + competitive-landscape
→ 供给方评估报告 + 备选供给方清单
     │
     ▼（供给方提供专利清单）
[模式C] 资产分级
patent.fetch（legal/family/citation）+ patent-asset-grading
→ 专利资产分级报告（S/A/B/C/D）+ IP边界核查
     │
     ▼（S/A级资产进入下阶段）
[模式D] 转化路径
tech-transfer-target-discovery
→ 引进路径推荐 + 内部承接建议
     │
     ▼
[模式E] 立项决策
rd-initiation-review + 证据链汇总
→ 立项决策包 + 尽职免责声明
```
