# Evidence Mapping Prompt — 证据链生成

## 任务
为每个重要分析结论生成可追溯的证据映射，确保所有结论有据可查。

## 证据链结构

每条证据链包含：
```json
{
  "claim_id": "C001",
  "claim_text": "结论文本（一句话）",
  "data_source": "MCP工具名 + 参数摘要",
  "data_value": "具体数值或统计结果",
  "supporting_patents": ["CN1234567A", "US9876543B2"],
  "evidence_strength": "高/中/低",
  "reasoning": "基于XX数据，通过XX推理得出此结论",
  "limitations": "如有限制，说明"
}
```

## 必须包含的证据结论

### 趋势类（至少 2 条）
- C001：申请趋势结论（必须来自全量统计）
- C002：近五年增长结论（必须来自全量统计）

### 竞争格局类（至少 2 条）
- C003：申请人集中度结论（来自申请人排名 MCP）
- C004：主要竞争者布局结论（来自申请人排名 + 代表专利）

### 地域布局类（至少 1 条）
- C005：技术来源国结论（来自地域分布 MCP）

### 质量影响力类（至少 1 条）
- C006：高影响力专利结论（来自最被引专利 MCP）

### 子技术机会类（至少 2 条，每个推荐方向一条）
- C007+：每个"推荐进入"子技术方向的机会依据

### 空白点类（至少 2 条）
- C010+：技术空白点依据

### 综合评分类（1 条）
- C020：综合推荐评分依据

## 证据强度评定标准
- **高**：直接来自 MCP 全量统计，可复现
- **中**：来自 MCP 部分统计或代表专利推理
- **低**：来自模型分析或间接推断，无直接数据支撑

## 证据链在报告中的使用
- 在 index.html 每个关键结论旁标注 [C001] 角标
- evidence.html 展示完整证据链表格
- evidence_mapping.csv 保存结构化数据
- intermediate_data.json 的 evidence_mapping 字段存储完整列表
