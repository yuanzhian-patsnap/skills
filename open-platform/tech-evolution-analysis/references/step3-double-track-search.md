## 零、v1 → v2 升级要点（双轨 → 四轨）

> **v1 局限**：检索只走专利。专利从研发到公开有 18+ 个月法定滞后，所以"3 年内可见"的预测，实质是在描述 2-4 年前已完成的研发——**回看包装成了预测**。学术论文比专利平均**领先 1-3 年**，是补偿这一滞后、捕捉前瞻信号的最低成本入口（也是 Step 7 黑天鹅/10 年预测的第一个"外部信号哨兵"）。
>
> **v2 改进**：每个组件的检索从 **本域/跨域 2 轨**升级为 **本域专利 / 本域论文 / 跨域专利 / 跨域论文 4 轨**。论文轨用 `source=paper` 检索，目标占比 ≥ 20%（下限非硬配额，见 §1.3）。

| 维度 | v1（双轨） | v2（四轨） |
|------|-----------|-----------|
| 轨道数 | 本域 + 跨域 = 2 轨 | 本域专利 / 本域论文 / 跨域专利 / 跨域论文 = 4 轨 |
| 桶数 | 6 组件 × 2 = 12 桶 | 6 组件 × 4 = 24 桶（零部件级同理翻倍） |
| 数据源 | 仅 `source=patent` | `patent` + `paper` 并轨 |
| 论文检索式 | 无 | **强制 `semantic` 主导**（见 §1.2 坑），keywords 收口 |
| 时间信号 | 仅公开日（滞后） | 论文 publication_date 提供 1-3 年前瞻 |
| 论文占比 | ≈0（仅偶发混入跨域桶） | 每轨目标 ≥ 20%，不足记 `paper_signal: sparse` |

## 一·补 论文轨检索规约（v2 核心新增）

### 1.1 为什么论文轨不能照搬专利检索式

实测教训（声学超材料域，2026-06）：论文轨若用「关键词 + 按引用排序」，回来的 Top 命中是《The rise of graphene》《QUANTUM ESPRESSO》《Coot 晶体学软件》这类**超高引但跨学科无关**的综述/工具论文。原因：

- 论文引用量级跨 **5 个数量级**（声学域顶刊论文才 100-200 引，而跨学科方法/工具论文动辄上万引）；
- 综述类、软件工具类论文引用畸高，会淹没真正相关的工程论文。

### 1.2 论文轨强制规约

| 规约 | 实施 | 理由 |
|------|------|------|
| **语义主导** | `search_strategy` 必须含 `semantic`，以 `semantic_query`（完整自然语言句）为主，keywords 仅收口 | 实测语义检索 5 篇全部精准命中（含目标域 EPFL 主动声学超材料综述） |
| **排序不靠绝对引用** | 论文轨**不**用 `sort=-cited_count` 作主排序；相关性优先，引用作辅助 | 避免跨学科超级综述污染 |
| **取摘要作证据** | 语义检索返回 `text` 摘要字段，留作 Step 5 标引证据（论文无 IPC，靠摘要定 tier） | 论文标引唯一可靠证据源 |
| **域内引用百分位** | `cited_min` 不照搬专利绝对值，按**同域百分位**校准（声学域顶刊才 161 引，门槛 ≈ 域内 P75） | 论文绝对引用与专利不可比 |

**论文轨检索示例**（C2 声腔 · 跨域论文轨）：

```python
patsnap_search(
  source="paper",
  search_strategy=["semantic","keyword"],
  semantic_query="active noise cancellation and sound absorption using "
                 "acoustic metamaterials for compact earphone applications",
  keywords=["acoustic metamaterial","noise control","earphone"],
  limit=30,
  # 注意：论文轨不设 sort=-cited_count；相关性优先
)
# 返回字段：id / title / publication_date / cited_count /
#          authors / org_names / doi / journal_name / text(摘要)
```

### 1.3 论文占比：目标下限而非硬配额

每轨论文目标占比 **≥ 20%**，但**不硬凑**：

- 先按相关性取够论文数；够 20% → 停；
- 不足 20% → 记 `paper_signal: sparse`，**这本身是一个信号**（说明该组件的演进不由学术驱动，而由工程/协议驱动，如 C6 蓝牙 SoC）；
- 不为凑比例把无关综述塞进来（宁缺毋滥，与专利轨克制原则一致）。

> **组件差异**：C1 发声单元 / C5 电池（材料物理驱动）论文前瞻信号强；C6 蓝牙 SoC（协议/工程驱动）论文少且偏理论 → 允许 sparse。

## 一·补2 Web 行研信号哨兵层（v3 新增 · 吸收自 black-swan-tech-radar）

> **为什么需要**：专利 + 论文仍是"结构化已布局信号"。真正的黑天鹅源头——资本异动、监管突变、跨界玩家、标准抢跑——**不在专利库里**。这层 Web 哨兵补足专利/论文盲区，是 Step 7 黑天鹅识别的第三类证据源（与论文轨并列）。

### 1.5 五类哨兵检索（每个分析必跑）

对目标产品 + 每个高价值组件，跑以下五类 Web 检索，产出 `web_signals.json`：

| 哨兵类型 | 检索式模板 | 捕捉的信号 | 喂给下游 |
|----------|-----------|-----------|----------|
| **资本哨兵** | `[技术领域] 融资 独角兽 风险投资 [年份]` | VC/PE 大额投资、新独角兽 → 资本提前下注的颠覆方向 | Step7 BSS 资本热度维 |
| **成熟度哨兵** | `[技术领域] Gartner hype cycle 技术成熟度` | Gartner 曲线阶段、TRL 信号 | Step7 TRL/时间档 |
| **标准哨兵** | `[技术领域] 标准 IEC/ISO/NIST 政策 技术路线图` | 标准立项/修订、强制政策 → 灰犀牛路径信号 | Step7 灰犀牛识别 |
| **监管哨兵** | `[技术领域] 监管 禁令 合规 认证` | 监管/事件驱动 → 黑天鹅触发器 | Step7 黑天鹅触发器 |
| **跨界玩家哨兵** | `[竞争对手/科技公司] 技术战略 最新布局 跨界` | 非传统玩家入场（互联网/航天/生物→本域） | Step4 颠覆者通道 + Step7 跨域迁移维 |

### 1.6 哨兵信号规约

| 规约 | 实施 |
|------|------|
| **证据可溯源** | 每条信号记 `source_url` + `date` + 一手/二手标记；严禁无源推测 |
| **置信度三档** | 🔴 多源交叉验证 / 🟡 部分支撑 / 🟢 单一来源（沿用 TechRadar 标注规范） |
| **降级策略** | Web 检索失败 → 降级为仅专利+论文，报告显式标注"行研信号缺失" |
| **不入评分池** | Web 信号**不进** Step 4 的 5 维打分（无专利属性），单独存 `web_signals.json`，仅在 Step 7 作黑天鹅/灰犀牛的交叉验证证据 |

> **定位**：专利轨答"谁已布局"，论文轨答"实验室验证了什么"，**Web 哨兵答"钱、标准、监管、跨界玩家在往哪动"**——三源交叉才能把黑天鹅识别从"专利内推测"升级为"多源印证"。

## 三、给 Step 4 的输入接口

Step 4 高价值筛选不需要再读原始 240 条记录，而是**复用 patsnap MCP 重打一次同样的 query**，加上更严格的 filter：

```python
# 示例：C3 ANC 域内高价值筛选
patsnap_search(
  search_strategy=["keyword","filter"],
  keywords=["主动降噪","ANC","feedforward","feedback","自适应降噪",
            "active noise cancellation","earphone"],
  filters={
    "ipc":["H04R","G10K"],
    "jurisdiction":["CN","US","JP","EP","WO"],
    "date_from":20180101,
    "date_type":"publication",
    "legal_status":["active"],   # 新增
    "cited_min":5                 # 新增
  },
  topk=50                          # 提到 50，以便筛选后留 20-30
)
```

**Step 4 筛选维度**（从原始池下钻到 ≈3000 件高价值池）：

| 维度 | 阈值（建议） | 工具实现 |
|------|--------------|----------|
| 法律状态 | active 或 granted | `filters.legal_status=["active"]` |
| 引用次数 | cited_min ≥ 5（高影响力）；ANC 这类热门域可提到 10 | `filters.cited_min` |
| 同族数 | 跨 ≥3 国布局（强战略意图） | 需 `patsnap_fetch` 取 family 模块后过滤 |
| 申请人质量 | Top 20 头部企业 + 顶尖院校白名单 | 用 `filters.assignees` 二次检索或后过滤 |
| 跨域权重 | 本域 70% / 跨域 30%（避免类比信号被淹没） | 分桶后按比例采样 |

每组件目标输出 ≈ 30-50 件高价值专利 → 6 组件合计 ≈ 200-300 件，作为 Step 5 TRIZ 标引输入。

> **论文桶衔接**：论文桶同样重打 query 给 Step 4，但走**论文专属权重模板**（仅 B 引用 + C 申请人/机构质量，关闭 A 同族 / D patsnap value / E 运营——论文无这些字段），`cited_min` 用同域百分位。详见 [step4-multi-dim-scoring.md](step4-multi-dim-scoring.md) §1.4。

## 四、本步骤局限 & 后续优化点

1. **专利检索时间窗仅 2018+**：跨域类比信号在 2010-2017 已出现的会被截断（如 GraphAudio 静电换能 2018 之前已多次申请）。Step 4 可对 C1/C2 跨域桶把 date_from 放宽到 20100101。
2. **论文轨标引需适配无 IPC/同族**（v2 升级）：论文轨已从"偶发混入跨域桶"升级为正式的本域/跨域论文轨。但论文没有 IPC、同族、法律状态字段，Step 5 标引须改用 `text` 摘要定 tier，Step 4 须走论文专属权重模板（见 step4 §1.4）。
3. **C5 跨域噪声大**：固态/柔性电池命中过宽，需在 Step 4 用 `cited_min ≥ 50` 强约束。
4. **论文轨语义检索成本**：每组件多 2 轨（本域论文 + 跨域论文），语义检索比关键词慢。**缓解**：论文轨 limit 可设 20-30（低于专利轨），靠语义精度而非召回量取胜。
5. **未做发明人流动追踪**：未来可对 Apple/Sony/Bose 核心 ANC 发明人做离职专利追踪——这是黑天鹅信号源。论文轨的 `authors` + `org_names` 字段为此提供了第二数据源（学界→产业流动）。
