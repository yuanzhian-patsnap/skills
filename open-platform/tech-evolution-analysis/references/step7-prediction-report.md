## 一、Step 7 的定位

前 6 步把 720 万件原始命中收敛成「四层级进化树 + 三类节点」的结构化数据。Step 7 在此基础上把**「数据」转化为「决策」**——回答四个问题：

1. **3 年内可见**：渐进升级路径，已有本域 v_max 节点直接对应
2. **5 年内可见**：子系统级跃迁路径，由空白点 + 类比点驱动
3. **10 年内可见**：形态级跃迁路径，由跨域突破点 + 黑天鹅信号驱动
4. **下一代旗舰概念形态**：把上述三档预测拼合到一个具体产品概念上

## 二、预测推理逻辑（v0-v11 节点 → 时间窗口）

### 2.1 三档预测的判别逻辑

| 时间窗口 | 判别规则 | 数据来源 | 置信度 |
|---------|----------|----------|--------|
| **3 年内可见** | 本域已有 `breakthrough=true` + 综合分 ≥ 60 + 头部 OEM 申请 | feature_points.json 的本域子集 | 高（已商用化候选） |
| **5 年内可见** | 跨域已有 `breakthrough=true` + 跨域专利占比 ≥ 50% + 同族 ≥ 3 国 | feature_points.json 跨域 + analogy_points.json | 中（产业链已动） |
| **10 年内可见** | 跨域 `breakthrough=true` 仅论文支撑 + 高引用 + 学术综述存在 | feature_points.json 跨域论文 + blank_points.json | 低（需突破多重门槛） |

> **TRL 轴作为时间窗口第二判据（v3 新增 · 吸收自 black-swan-tech-radar）**：tier（TRIZ 节点）答"往哪走"，**TRL 答"多快到"**，二者正交互补。用 TRL 校正纯路线周期映射：
> - **TRL 7-9**（产品化/市场化）→ 收敛到 **3 年档**；
> - **TRL 4-6**（技术验证/原型）→ **5 年档**；
> - **TRL 1-3**（基础研究，多为论文 `leading_signal`）→ **10 年档**。
> - 当路线周期映射（§2.3）与 TRL 判断冲突时，**以 TRL 为准**（实证成熟度优于经验先验）。TRL 取证来自 Step3 成熟度哨兵 + 论文/专利证据。

### 2.3 路线 → 时间窗口的经验映射

> **前置原则（v3 新增 · 方向基准）**：预测"下一节点"必须**沿该路线的内在方向**取（见 [step5 §3.1 路线方向语义](step5-triz-labeling.md)）。`next_node` = 当前节点沿**该路线方向**的下一个字典节点，而非"更复杂的节点"。
> - **增益型路线**（3/4/5/6/7/8/10/11）：下一节点 = 更结构化 / 更智能 / 更微观；
> - **裁剪型路线**（路线 2、路线 1 末端）：下一节点 = **更简、更集成、部件更少**（如"多功能→单功能极限→自服务"）。此时预测出的下一代形态应当是**做减法的形态**，不得误判为"功能不足"而强推复杂化。
> - **判据**：若某组件的本域主干落在裁剪型路线，其 5/10 年预测应描述"如何用更少实现同等/更优"，`transition_signal` 写成降本/集成/去部件，而非堆叠新模块。

不同 TRIZ 路线的演化速度不同，可作为推理时间窗口的辅助先验：

| 路线 | 演化速度 | 典型周期 | 备注 |
|------|----------|----------|------|
| 路线 6 系统可控性 | 快 | 1-3 年 | 软件/算法主导 |
| 路线 7 频率匹配 | 中 | 3-5 年 | 算法 + 部分硬件 |
| 路线 1 单-双-多 | 中 | 3-5 年 | 集成度提升 |
| 路线 5 内部结构 | 中-慢 | 5-7 年 | 结构 + 材料 |
| 路线 3 向微观 | 慢 | 7-12 年 | 化学/物理体系跨代 |
| 路线 8 动态化（柔性） | 慢 | 7-12 年 | 制造工艺重构 |
| 路线 11 复杂线性 | 中 | 3-7 年 | 阵列 + 算法 |

> 推理时把组件当前 tier 加上路线对应周期，与「跨域 max tier - 本域 max tier」的 gap 差距匹配，得到时间窗口估算。

## 三、灰犀牛 vs 黑天鹅信号识别（v2 升级）

> **v1 局限**：原版只有"黑天鹅信号"一节，但 PLC 案例（2026-06）暴露问题——把 7 个突变全部归为黑天鹅，实际上其中 4 个（软 PLC 替代 / PAC+NPU / 5G uRLLC / 中国信创）是看得见的庞然大物（灰犀牛），趋势已定，只是企业准备不足；真正的黑天鹅只有 3 个（量子破解时点 / LLM 跨过 SIL3 认证 / 重大 OT 攻击）。
>
> **v2 改进**：把"突变信号"分成两类，监控频率与战略动作完全不同。

### 3.0 分类框架（必须在 §3.1 之前先做）

| 维度 | 灰犀牛 Gray Rhino | 黑天鹅 Black Swan |
|------|-------------------|-------------------|
| **发生概率** | HIGH（趋势已定） | LOW（事件触发） |
| **可预测性** | 时间窗清晰 | 触发点 / 时点不可预测 |
| **专利信号特征** | 跨域已 v9+ 等待植入 / 标准化在路上 / 头部厂商已布局 | 本域 ≤v6 + 监管/事件触发 / 非线性跨过认证或物理门槛 |
| **监控频率** | 半年 — 年度（路径监控） | 季度 — 事件触发（信号监控） |
| **战略动作** | 纳入路线图 + 提前布局 | 设置触发器 + 准备应急方案 |
| **典型表述** | "5 年内必将发生 X" | "X 一旦发生立即冲击全行业，但时点不可预测" |

### 3.0.1 BSS / TDI 量化打分（v3 新增 · 吸收自 black-swan-tech-radar）

> **解决 Q9/Q11**：原版灰犀牛/黑天鹅只有定性"≥3 项/≥2 项"判据，阈值靠拍脑袋。引入两个量化骨架，把定性规则升级为可加权、可回测的评分。每个突变候选先算这两个分，再做灰犀牛/黑天鹅归类。

**BSS 黑天鹅信号强度（满分 10）—— 答"信号有多强"**

| 维度 | 分值 | 数据源 |
|------|:----:|--------|
| 专利增速指数（近 3 年 CAGR > 30%） | 2 | Step5 标引池年度分布 |
| 跨域迁移度（来自非传统行业） | 2 | Step5 `analogy_candidate` 跨大系统 + Step3 跨界玩家哨兵 |
| 资本热度（VC/PE 大额投资） | 2 | Step3 资本哨兵 |
| 学术前沿度（顶尖机构论文密度+增速） | 2 | Step3 论文轨 `leading_signal` |
| 监管/标准驱动 | 1 | Step3 标准+监管哨兵 |
| 成本曲线临界 | 1 | Step3 成熟度哨兵 + 硬指标 |

**TDI 技术颠覆指数（满分 100）—— 答"颠覆有多大、多急"**

```
TDI = 替代潜力(0-30) + 市场冲击度(0-30) + 时间窗口(0-20) + 生态成熟度(0-20)
预警等级：观察(0-25) / 关注(26-50) / 预警(51-75) / 紧急(76-100)
```

> **两分的分工**：BSS 度量"信号强度"（用于排序哪些候选值得关注），TDI 度量"冲击量级+紧迫度"（用于定预警等级与资源投入）。二者均要求**每维有具体数据支撑，不得仅凭定性描述**。

**量化分如何接入灰犀牛/黑天鹅归类**（替代纯计数判据）：

| 归类 | 量化条件 | 与定性规则的关系 |
|------|----------|------------------|
| **灰犀牛** | TDI ≥ 51（预警/紧急）**且** 路径清晰（时间窗 ≤4 年 / 标准化在路上） | 量化门槛 + §3.1 定性条件双轨确认 |
| **黑天鹅** | BSS ≥ 6（信号强）**且** 跨物理/数学/监管门槛 + 时点不可预测 | 量化门槛 + §3.2 非线性判据双轨确认 |
| **双重身份** | 同时满足 → 按 §3.3 分别归类 | — |

> **阈值非硬编码**：TDI 51 / BSS 6 是初始建议值，须用历史案例验证集校准（见 §七局限 6 回测台账）。

### 3.1 灰犀牛识别规则

灰犀牛 = 满足以下 ≥3 项：

| 必要条件 | 数据字段 |
|---------|----------|
| 跨域已有 v9+ 突破点（路径清晰） | feature_points cross v9+ ≥2 |
| 标准化路径明确（IEC/ISO/NIST 已立项或修订） | **Step3 标准哨兵自动取证** + 人工复核（原手工标注升级） |
| 头部厂商已提交申请（≥2 家头部 OEM） | applicant_top_2 ≥2 |
| 商业产品已发布（≥1 家已上市） | **Step3 资本/跨界玩家哨兵自动取证** + 人工复核（原手工标注升级） |
| 时间窗清晰（窗口跨度 ≤4 年） | predicted_window |

### 3.2 黑天鹅识别规则（v1 规则 + 强化非线性判据）

黑天鹅 = 灰犀牛规则之外，且满足以下 ≥2 项：

| 必要条件 | 数据字段 |
|---------|----------|
| 跨过物理 / 数学门槛（如量子破解、SIL 认证、形式化证明） | 手工标注（物理判断不可自动，保留人工） |
| 监管 / 事件驱动（非自然演化驱动） | **Step3 监管哨兵自动取证** + 人工复核（原事件类型标注升级） |
| 跨大系统类比（A 系统已有 + B 系统未有） | analogy_candidate 跨大系统 |
| 涉及形态级路线（3/8/11 或 tier ≥9） | route ∈ {3,8,11} 或 tier ≥9 |
| 触发后冲击窗口 ≤12 个月（年度变量瞬间塌缩为月度） | 影响窗口手工标注 |

> **Q10 缓解**：可自动取证的判据（标准化进度、产品发布、监管事件）已接 Step3 Web 哨兵自动拉取 + 人工复核；真正需人工的只剩"是否跨物理/数学门槛"这类专业判断——这才是分析师价值所在，配 §3.2.1 判断模板使其可审计。

### 3.3 同一议题可同时具备两种身份（重要）

许多重大议题在不同切面下身份不同——必须明确切分，分别归类：

| 议题 | 灰犀牛切面 | 黑天鹅切面 |
|------|------------|------------|
| **PQC 抗量子** | NIST 已 FIPS 标准化 + IEC 62443-4 修订 + 嵌入式厂商已 v9 → 5 年内强制升级 | 量子计算机什么时候真能跑 Shor 破解 RSA-2048 不可预测，提前 2-3 年发生即从"有序升级"变"恐慌替换" |
| **GenAI-IDE** | Rockwell+Siemens+ABB 三家齐推辅助编程 → 5 年内全面落地 | LLM 直接生成可通过 SIL3 认证的代码是非线性跨越，节奏由 LLM 能力 + 责任认定博弈决定 |
| **国产替代 / 信创** | 清单 + 政策 + 产业链已动 → 5 年内中国市场份额重排 | 极端情况下若出现禁运升级或重大供应链事件，时间线压缩到月度 |

> **方法论提醒**：识别这类双重身份的议题最有价值——灰犀牛切面纳入路线图，黑天鹅切面作为触发器监控。

## 四、下一代旗舰概念形态

### 4.1 拼合规则

把 6 组件的 5 年 / 10 年路径拼合到一个具体产品概念上：

```
for each component in [C1..C6]:
    选取该组件 5yr 与 10yr 预测的"主驱动节点"
    沿该组件主干路线的【内在方向】取节点（增益型→更强；裁剪型→更简）
    汇总成「下一代旗舰」的组件级描述
合成产品定义：
    描述该形态的「核心价值」（不是功能升级，而是品类定义重塑）
```

> **方向校验（v3 新增）**：拼合前先看每个组件主干落在增益型还是裁剪型路线。**裁剪型组件不得在概念形态里被描述成"加更多模块"**——它的下一代形态是"用更少做到同样/更好"。一份健康的概念形态通常**增益与裁剪并存**（如发声单元走微观增益、声腔走结构增益、而整机装配/部件数走裁剪），而非所有组件一致堆复杂度。若拼合结果是"每个组件都更复杂"，多半是误用了方向，需回查。

### 4.2 概念形态条目模板

```markdown

  └── 内嵌压电微结构按场景实时调谐谐振频率
- **ANC 模块**：物理建模 + 局部静区生成（路线 5 v5 + 路线 11 v4）
  └── 麦阵列 + 主动声腔 + 算法三模态联动
- **麦克风阵列**：8-12 mic 仿生球面阵（路线 1 v3 + 路线 9 v4 + 路线 11 v4）
- **电池**：半固态柔性贴片电池（路线 3 v9 + 路线 8 v3）
- **蓝牙 SoC**：UWB + BLE 双链路 + 光学辅助（路线 1 v3 + 路线 3 v10 + 路线 7 v4）

### 整体形态定义

```

### 4.3 替代形态分支（多概念候选）

为了避免单一概念形态的过拟合，Step 7 应同时输出 **2-3 个替代形态分支**：

| 分支 | 拼合策略 | 适用场景 |
|------|----------|----------|
| **保守路径** | 全部用 5 年预测 | 若行业进展低于预期 |
| **激进路径** | 全部用 10 年预测 | 若头部突破加速 |
| **黑天鹅路径** | 5 年预测 + 1-2 个黑天鹅信号 | 若颠覆性技术意外商用 |

## 五、报告产物（多风格 HTML）

### 5.1 报告章节标准结构

### 5.3 一致性校验（LR-01 至 LR-08）

| LR-03 | 类比点 tier 差 ≥ 2 | 标 WARN |

## 七、本步骤局限与缓解

1. **3-5-10 年时间窗口的经验性** — 路线 → 周期映射是先验，不同行业需校准。**缓解**：Step 7 输出附「时间窗口校准说明」，列出各路线的本项目周期假设；用户可调整。
2. **概念形态的过拟合风险** — 单一概念形态可能给出过强的确定性。**缓解**：3-2-3 节强制输出 2-3 个替代分支（保守/激进/黑天鹅）。
3. **黑天鹅监控的执行成本** — 季度 / 月度监控需要持续的 patsnap 订阅。**缓解**：黑天鹅清单挂接 `monitor_priorities` 字段，可由 cron 任务半自动化执行。
4. **报告生成与底层数据脱节** — 报告 HTML 一旦生成就静态化，与 evolution_trees.json 的更新脱节。**缓解**：报告生成器接受 `--auto-refresh` 参数，跑全流水线 → 重新拉数据 → 重新渲染。
5. **三种风格的内容一致性** — 模板差异可能导致同一字段在不同风格下措辞偏差。**缓解**：内容生成与样式分离，所有文字内容由同一份 `report_content.json` 提供，模板只负责样式。
