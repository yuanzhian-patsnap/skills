## 一、系统拆解逻辑

按 **执行 / 控制 / 动力 / 传输** 四类进行第一层划分（这是大系统的划分依据，不是标签），然后逐级下钻：

```
产品 → 大系统 → 子系统 → 组件 → 关键零部件
```

- **大系统**：按功能在产品中的角色拆出 4 个（执行 / 控制 / 动力 / 传输）
- **子系统**：在大系统下按物理归属再拆
- **组件**：可独立检索专利的最小功能单元
- **关键零部件**：决定该组件性能上限的核心物理元素（如振膜决定 C1 音质上限、电解质决定 C5 能量密度上限）

### 关键零部件的判别原则（四独立性）

是否构成「关键零部件」，按四个独立性逐项打分（各 1 分，总分 4 分保留 / 1-2 分合并 / 0 分删除）：

1. **物理学独立**：是否被独立的物理学定律约束（声学/电磁学/电化学/MEMS 工艺等）
2. **矛盾独立**：是否承载独立的核心技术矛盾（如振膜「灵敏度↔刚性」、电解质「电导率↔稳定性」）
3. **接口独立**：是否有稳定可替换的物理接口（接口越稳定，演化越独立 — Baldwin & Clark《Design Rules》）
4. **生态独立**：是否有独立的产业链玩家与研究社群（如电解质由电池厂商推进，与发声单元厂商不重叠）

四个都「是」→ 保留为关键零部件 / 1-2 个「否」→ 合并到父组件 / 全否 → 删除（装配件）

> v2 初版列出 23 件，按四独立性重审后收敛为 v3 的 **13 件**（删除 10 件冗余）。

关键零部件层是必须有的 — 例如振膜的材料演进（**纸盆 → PET → PEEK → LCP → 陶瓷 → 石墨烯**）在路线 4（表面特性）和路线 3（向微观）上各自有独立轨迹，不能淹没在「同轴双动圈/平板振膜」的粗描述里。

## 三、SVOP 写作规范与组件级 SVOP（v4）

### 3.1 写作规范（5 条）

> 格式：`Subject(主体) → Verb(动作) → Object(客体) → Parameter(客体被改变/保持的物理量)`
> 核心目标：通过 V+O 的**上位化**（一般化）扩大跨域召回范围

| 字段 | 规范 |
|------|------|
| **S 主体** | 保持具体（这是分析对象本身） |
| **V 动作** | 上位到「跨产品族通用动词」，剥离行业术语（✗ 振动辐射 → ✓ 致动） |
| **O 客体** | 上位到「物理本体」，剥离材料/介质特指（✗ 空气声波 → ✓ 弹性介质） |
| **P 参数** | 必须是**客体在动作下被改变或被保持的物理量**，不是主体的属性（✗ 杨氏模量GPa → ✓ 声压级dB） |
| **多功能取主** | 只取决定该件存在意义的**主功能**，次功能丢弃；多客体只取主客体 |

**上位化的"度"**：判别标准是「**能否拉到 3 个以上完全不同的产品族**」。
- 太具体（"振动 空气"）→ 仅扬声器域 ❌
- 太抽象（"作用 物质"）→ 召回过宽全是噪声 ❌
- 合格（"致动 弹性介质"）→ 扬声器/超声换能/压电致动/MEMS发声/医学震波 ✓

**多功能示例**：振膜其实有两个功能——主功能「致动声波」+ 次功能「隔离前后腔空气」。按规范只保留主功能，避免把密封件误拉入检索。

### 3.2 组件级 SVOP（6 条 · v4 重写）

| 编号 | 组件 | 大系统 | S → V → O | P（客体被改变量） | 跨域 V+O 锚点 |
|------|------|--------|-----------|-------------------|---------------|
| C1 | 发声单元 | 执行 | 换能器 → 转换 → 能量形式（电→机械→声） | 声压级dB / 频率响应Hz | convert energy form / 能量形式转换 |
| C2 | 声腔 | 执行 | 腔体 → 调谐 → 波动场 | 共振频率Hz / Q值 / 频响平直度 | tune wave field / 调谐 波动场 |
| C3 | ANC模块 | 控制 | 主动控制器 → 抵消 → 干扰场 | 残余强度dB / 抑制带宽Hz | cancel interference field / 抵消 干扰场 |
| C4 | 麦克风阵列 | 控制 | 阵列 → 空间采样 → 场分布 | 指向性dBi / 空间分辨率° | spatially sample field / 空间采样 场分布 |
| C5 | 电池 | 动力 | 储能装置 → 储释 → 能量 | 能量密度Wh/L / 循环寿命次 | store and release energy / 储释 能量 |
| C6 | 蓝牙SoC | 传输 | 收发器 → 无线传递 → 信息流 | 吞吐率bps / 端到端延迟ms | wirelessly relay information / 无线传递 信息流 |

> **v3 → v4 关键修正**：原 P 字段中「频响20Hz-20kHz / 灵敏度dBV/Pa / 容量mAh / 码率kbps」都是**主体（设备）的性能指标**，已改为**客体（声场/能量/信息流）被改变或保持的物理量**。这一修正使得 V+O 检索能正确匹配跨域同功能技术（如能量储释会拉到飞轮储能、压缩空气储能、超级电容；无线传递信息流会拉到光通信、UWB、可见光通信）。
