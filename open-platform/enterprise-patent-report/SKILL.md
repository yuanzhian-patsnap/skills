# 企业专利战略研究报告生成技能

## 技能定位

为企业高管（CEO/总经理级别）生成一份**有实际数据支撑的专利战略研究报告**。报告采用常规报告排版（非PPT翻页），从上到下连续滚动阅读，所有核心主张均绑定专利号或文献引用，未能核实的数据一律标注 `Unverified`。

报告服务于两个目标：
1. **让企业高管快速理解自身专利资产与竞争位置**
2. **引出智慧芽三模块服务方案（导航布局 + TRIZ降本攻关 + FTO风险防控）**

---

|------|------|---------|---------|

## 分步执行流程

### Step 0：信息收集（开始前）

向用户收集以下信息：
- **企业名称**（中英文，用于PatSnap申请人过滤）
- **所在行业**（用于行业检索词设计）
- **主要技术方向**（3-5个关键词）
- **已知竞品企业**（国内+国际各2-3家）
- **目标读者**（职位，如：总经理/CEO/董事会）
- **智慧芽服务方案附件**（要求用户上传以下6份PPT）：
  - 储能/行业类客户案例
  - 专利布局规划解决方案
  - 专利风险防控解决方案
  - 研发情报赋能方案
  - TRIZ+AI助力技术与产品研发攻关
  - TRIZ+AI助力研发与产品专利规避

### Step 1：PatSnap多维检索（制作前必做）

为每个核心主张执行检索，建立证据库：

```
检索1：企业自身专利（申请人过滤）
  → 获得：总件数、发明专利占比、核心专利列表

检索2：行业全景（宽泛关键词，topk=30）
  → 获得：行业热点技术方向、头部申请人排名

检索3：竞品专利（每家竞品单独检索，topk=15）
  → 获得：竞品技术方向、差异化对比依据

检索4：核心应用场景（场景关键词，topk=20）
  → 获得：场景专利佐证、行业落地案例

检索5：高被引文献（关键技术性能对比，topk=15）
  → 获得：第三方文献对性能参数的独立验证
```

### Step 2：证据绑定原则

每个核心主张必须绑定证据，否则标注 `Unverified`：

| 主张类型 | 要求的证据 |
|---------|---------|
| 企业专利数量 | PatSnap检索命中件数（申请人过滤后） |
| 联合客户案例 | 联合申请专利（申请人栏含客户名称） |
| 性能对比数据 | 高被引综述文献（引用次数≥20次） |
| 竞品技术定位 | 竞品代表专利+摘要技术方向描述 |
| 市场规模数字 | 若无第三方研报，标注Unverified |
| 政策驱动内容 | 相关政策背景专利或标准文件 |

### Step 3：各部分制作要点

#### Part 1 摘要
- 封面标题：**不含读者姓名或职务称谓**（如"王总快速抓重点"等个人化文字一律删除，保持客观报告风格）
- 副标题使用"核心判断"/"战略摘要"等通用表述
- 四象限布局：左上定位（蓝）、右上优势（绿）、左下行业地位（琥珀）、右下结论（紫）
- 顶部封面含4个核心数字徽章（专利件数、寿命年限、性能指标、落地案例数）
- 每个象限底部附1-2条专利链接作为基础证据

#### Part 2 公司概况
- 资质背书卡片：学术背景/国家级奖项/专精特新/高新技术（逐条列示，不拔高）
- 产品规格表：至少4行，涵盖不同功率/容量/响应时间梯度
- 联合申请专利盒：用于佐证客户落地案例（申请人栏含客户名即为强证据）
- 产能交付：双基地格局+年产能+标杆客户展示

#### Part 3 核心竞争力
- 性能对比表（7-8维度，企业产品 vs 对标技术）
- 绿色高亮"领先"，标注具体倍数
- 文献证据盒：至少2篇高被引综述支撑技术性能主张
- 标杆案例量化：核心指标前后对比，绑定联合申请专利

#### Part 4 竞争格局
- 国内竞品表：选3-4家，按专利检索结果客观填写技术方向，不夸大差距
- 国际竞品卡片：技术路线错位分析（如Amber 8kW vs 自身 4MW = 互补非竞争）
- 跨技术路线对比：结论聚焦优势场景，不贬低其他路线

#### Part 5 市场机会
- 政策驱动：列3条政策，附相关专利佐证
- 应用场景优先级表：按ROI/政策支持/已有案例排序
- 市场规模数字：若无来源一律标注 `Unverified`

#### Part 6 核心风险
- 三大风险：**成本/认知/竞争**（此三类对大多数储能/硬科技企业通用）
- **成本风险必须点明"技术矛盾"属性**：规模化降本往往是多维约束下的技术矛盾（如降成本vs保精度），为后续Part 8 TRIZ方案埋下伏笔
- 每条风险：发生概率+影响程度+当前管控状态+具体应对措施
- 风险评分矩阵（3×4表格）置于页尾

#### Part 7 战略建议
- 三条建议对应：技术迭代/规模化/生态合作
- 每条建议含：具体目标（含年份）+执行步骤+已有专利基础
- 三步走路线图：Phase1筑基/Phase2扩张/Phase3领跑，含营收估算（标Unverified）

#### Part 8 智慧芽方案（核心章节，需重点打磨）

> ⚠️ **关键原则：痛点视角必须是行业共性，绝不指向具体企业**
> 主语始终用"储能企业普遍"/"行业通常"/"大多数科技企业"，不写"该企业"或客户名称。

##### 三大行业共性痛点（来自附件原文）

**痛点一：布局结构散**
> 企业普遍重研发、轻布局。专利随研发自然生长，缺乏战略规划，核心技术存在保护空白区，PCT出海布局严重滞后于研发节奏。

**痛点二：保护力度弱**
> 申请量增长但质量参差不齐；权利要求写得过窄，发明专利占比偏低；创新点转化效率低，随项目结束而流失。

**痛点三：管理方式粗放**
> 专利资产沉睡，研发与IP之间信息壁垒严重；无系统竞对动态监控；IP团队依赖外部顾问，能力无法内化沉淀。

##### 三模块服务方案

**模块一：专利导航布局报告**（来自《储能类客户案例》附件）

六章导航报告结构：
1. **理清家底** — 企业现有专利地图，覆盖度诊断
2. **看清战场** — 行业技术分布，热点与空白
3. **破解对手** — 竞品技术路线拆解，差异化布局依据
4. **布网** — 核心技术点挖掘，多维度专利布局方案
5. **抄作业** — 行业标杆布局策略参考（如宁德时代2015-18年布局数据）
6. **拍板清单** — 可立即执行的专利申请优先级清单

授之以渔：企业IP团队掌握导航方法论，不依赖顾问驻场。

**模块二：TRIZ+AI降本攻关**（来自两份TRIZ附件，此模块承接Part 6成本风险结论）

> **方案与报告结论的衔接点**：报告第6部分指出成本压力是首要风险，而降本往往涉及典型技术矛盾（如"既要降低制造成本，又要保持核心性能指标"）——这正是TRIZ方法论的核心应用场景。

TRIZ攻关五步流程（来自附件Slide 11/19）：
1. **功能分析** — 拆解系统组件与功能关系
2. **因果分析** — 定位问题根因
3. **裁剪** — 简化冗余结构，降本基础
4. **矛盾构建** — 将问题转化为技术/物理矛盾
5. **方案生成** — 基于TRIZ 40条发明原理输出20+创新方案

效率对比数据（来自附件Slide 20）：
- 传统工程师：约2个方案 / 30天
- TRIZ+Eureka AI：约20个方案 / 3天
- 提效约**10倍**

> ⚠️ 价格信息：**难题攻关工坊起价12万元**（3天/题）

**模块三：FTO风险防控 + TRIZ规避设计**（来自《专利风险防控》附件）

FTO风险数据（来自附件原文，非自编）：
- 近年美国专利诉讼案件数：**1173起**
- 平均判赔额：**1.67亿元人民币**
- 出海企业若无FTO预警，一旦侵权代价极高

规避设计三步法（来自《专利规避》附件Slide 7）：
1. **技术拆解** — 分解竞品专利权利要求
2. **组件重组** — 识别可替换/可绕过的技术要素
3. **跨领域检索** — 借助Eureka AI在非本领域寻找替代方案

##### 授之以渔：9大AI Skill（来自附件AI能力清单）

### Step 4：HTML报告规范

#### 排版规范
```
- 报告格式：常规报告（非PPT翻页，无翻页控制栏，无幻灯片底部圆点）
- 阅读方式：从上到下连续滚动
- 顶部导航：固定导航栏，8个章节锚点跳转，滚动时自动高亮当前章节
- 最大宽度：960px，居中，左右padding 24px
- 顶部padding：80px（避免被导航栏遮挡）
- 打印优化：Ctrl+P可导出完整PDF，导航栏@media print自动隐藏
```

#### 9种核心组件
1. **封面卡**：深色渐变背景+光晕装饰+核心数字徽章
2. **四象限卡**：2×2 grid，顶部3px渐变色条
3. **证据盒（专利）**：蓝色左边框，`#eff6ff`背景，含可点击PatSnap链接
4. **证据盒（文献）**：绿色左边框，`#f0fdf4`背景，含被引次数

7. **指标数字卡**：渐变色大数字，适合展示核心KPI
8. **Unverified标签**：`#fef3c7`黄底，行内显示，标注未核实数据
9. **TRIZ矛盾卡**：`--triz-light`背景，青绿左边框，用于展示技术矛盾与攻关方向

## 常见错误与纠正

| 错误 | 正确做法 |
|------|---------|
| 把行业通用痛点归因给具体企业 | 始终用"行业普遍"/"企业通常"作主语 |
| 用PPT翻页格式制作报告 | 常规报告：连续滚动，无翻页控制栏 |
| 市场规模数字无来源直接写入 | 先写数字后加`Unverified`标签 |
| 合并时CSS变量冲突 | 所有部分统一使用同一套变量体系，含`--triz`/`--triz-light` |
| 证据盒只写专利号不写链接 | 必须附PatSnap全文URL，方便读者验证 |
| 竞品性能直接复用用户说法 | 须用PatSnap检索竞品专利后客观描述技术方向 |
| Part 8方案与报告结论脱节 | 三模块必须承接Part 6风险结论：成本→TRIZ/布局→导航/出海→FTO |
| TRIZ内容自行编造 | 严格来自《TRIZ+AI》两份附件原文，含效率数据和流程图 |
| 摘要标题含读者姓名 | 删除所有个人化文字，保持客观报告风格 |
| 难题攻关工坊价格写错 | 起价**12万元**（3天/题），非7.5万 |
