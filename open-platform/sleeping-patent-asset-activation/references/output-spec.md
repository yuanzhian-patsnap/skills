# 客户成果物规范

## 交付策略

真实分析默认必须交付可独立打开的客户成果物，而不是只输出聊天文本。

- 默认格式：HTML 报告。
- 增强格式：本地环境可稳定渲染时，同时导出 PDF。
- 用户指定 Word、Excel、PPT 或 Markdown 时，按指定格式生成；其中 Excel 适合资产主表和买方触达表，Word/PDF 适合正式报告。
- 用户没有指定格式但已经提供可分析对象时，不追问格式，直接按 HTML 默认交付。
- 用户只给了模糊意图或缺少最小输入时，先追问一次：“本次要生成客户版 HTML/PDF 报告，还是先做文本预览？若不指定，我将默认生成 HTML 报告。”
- 最终回复必须包含成果物路径、报告内容摘要和仍需人工复核的事项。

## 推荐结构

1. 封面摘要：资产总数、可外推资产包、90 天线索、管理层决策。
2. 资产盘活漏斗：从资产池到 A/B/C/D/E 分层。
3. 高潜力资产清单：排名、评分、证据强度、推荐路径。
4. 重点资产招商卡：每个 A 类或重点 B 类资产一张。
5. 买方匹配与证据链：潜在买方、匹配理由、来源等级。
6. 补证和风险清单：权属、法律状态、技术证据、披露限制。
7. 30/60/90 天行动计划：动作、责任人、交付物、验收标准。
8. 方法说明和免责声明：MCP、公开来源、评分口径、非法律意见。
9. 专利详情链接说明：专利号链接来源、数据库访问权限和未链接清单。

## 视觉规范

风格采用企业服务优先的瑞士国际主义极简科技风。保持克制、清晰、可信，适合管理层、法务、研发、商务和 IP 团队直接阅读。

- 首屏直接给结论、分层数量和推荐动作。
- 使用中性色、网格系统、强对齐、留白和少量高对比强调色。
- 推荐色：深墨色、冷灰、专利蓝、运营绿；橙色只表示待补证；红色只表示阻断风险。
- 使用表格、评分条、证据标签、行动时间轴和资产卡。
- 每个结论旁边显示证据等级或来源类型。
- 报告中的专利公开号/公告号默认做成可点击链接，优先指向智慧芽 Eureka 详情页；链接样式应清晰但克制，不破坏表格可读性。
- 外部版隐藏敏感技术细节；内部版可附完整证据表。

## 默认表格字段

### 资产盘活主表

排名、资产包名称、关联专利/商标/软著/项目、核心应用场景、评分、分层、推荐路径、潜在买方类型、证据强度、交易洁净度、下一步动作、责任人。

关联专利字段中的公开号/公告号应优先链接到专利详情页。若使用智慧芽 Eureka 链接，URL 格式为 `https://eureka.zhihuiya.com/view/#/fullText'figures/?patentId={patent_id}`，其中 `patent_id` 由 `global_core_patent_database` 或 `advanced_patent_search` 返回；至少抽样使用 `patsap_patent_search.patsnap_fetch` 校验 1 条链接可被 MCP 识别。

### 单项资产招商卡

一句话卖点、客户痛点、技术方案、证据链、适用行业、买方画像、合作模式、首轮外推话术、NDA 前可披露材料、NDA 后材料、待补证事项。

### 买方触达表

买方名称或类型、匹配理由、证据来源、触达优先级、禁触达或注意事项、触达话术、预期下一步。

## 免责声明

结论仅用于商业化筛选和运营决策参考，不构成法律意见、正式资产评估报告、交易承诺或投资建议。正式交易、许可、质押或诉讼前，应由律师、评估机构和业务负责人复核。

专利详情页链接可能需要已登录账号或机构数据库权限；外部分发时，应避免把数据库内不可公开内容直接复制到公开报告中。
