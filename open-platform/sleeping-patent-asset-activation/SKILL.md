# 沉睡专利资产盘活官

## 快速定位

把“账面上的知识产权”转成“可以推进的业务机会”。不要只按专利数量或单件估值排序；要把专利、商标、软著、项目、样机、客户验证、市场需求和潜在买方组织成可许可、可转让、可联合开发、可质押融资或可清理放弃的资产包。

## 必用数据与 MCP

优先使用这 6 个 MCP。若工具未直接暴露，先用 `tool_search` 搜索并加载对应 MCP。

| MCP | 主用途 | 主口径 |
|---|---|---|
| `corporate_intangible_assets` | 企业无形资产档案、商标/软著/集成电路布图/植物新品种、潜在买方 | 非专利资产和买方初筛 |
| `patent_valuation_scorecard` | 单件专利价值评分、技术/市场/战略/经济/法律维度 | 价值分和维度解释 |
| `patent_monetization_valuation` | 法律状态、交易洁净度、家族、权利要求/摘要/说明书翻译、商业化统计 | 交易可运营性 |
| `patsap_patent_search` | 语义/关键词探索、专利/论文混合检索、Markdown 正文读取、详情页 URL 校验 | 找到并读懂候选证据 |
| `advanced_patent_search` | 精确检索式、数量统计、字段分布、相似专利、申请人扩展 | 技术拥挤度和统计口径 |
| `global_core_patent_database` | 权利要求、同族、被引、PDF、摘要图、翻译、专利 ID 等底层事实 | A/B 类资产事实核验与详情页链接 |

后三个搜索类 MCP 不冲突：`patsap_patent_search` 做探索层，`advanced_patent_search` 做统计层，`global_core_patent_database` 做核验层。详细边界见 `references/mcp-boundaries.md`。

## 报告交付策略

真实盘活任务默认必须产出客户可独立打开的成果物，不要只停留在文本结果。除非用户明确说“先给文字版”“不需要文件”“只要结论”，否则按以下规则执行：

- 用户指定 HTML/PDF/Word/Excel 时，按指定格式生成。
- 用户没有指定格式，但已经给出公司、专利号、台账或技术方向等可分析对象时，默认生成 HTML 报告；若本地环境可以可靠渲染，再同步导出 PDF。
- 用户只给了非常模糊的意图，先做一次简短追问：“本次要生成客户版 HTML/PDF 报告，还是先做文本预览？若不指定，我将默认生成 HTML 报告。”
- 若缺少主体、专利清单或技术方向，先追问补齐最小输入；不要把一段文本分析当成最终交付。
- 最终回复必须给出成果物路径或链接，并用 3-5 行说明报告包含什么、哪些结论需要复核。

## 工作流

1. 明确任务类型：外推招商、年费清理、质押融资、技术转移、并购尽调或组合运营。说明本次评分权重是否需要调整。
2. 建立资产池：归一主体名称、历史名称、子公司、院系、成果转化公司和发明人团队。用 `corporate_intangible_assets` 补齐商标、软著、集成电路布图、植物新品种和潜在买方线索。
3. 聚类成资产包：按技术问题、应用场景、发明团队、项目编号、同族、引用、商标/软著/产品名和样机/测试材料分组。资产包必须有外部可读名称，不要只列专利标题。
4. 做交易洁净度门禁：用 `patent_monetization_valuation` 查法律状态、家族、权利要求和翻译文本；必要时用 `global_core_patent_database` 复核同族、引用、PDF。权属不清、共有权未处理、已质押/许可、临近失效或敏感披露限制的资产不得进入 A 类。
5. 做技术与价值评估：用 `patsap_patent_search` 探索语义样本和正文，用 `advanced_patent_search` 统计相似专利与申请人分布，用 `global_core_patent_database` 核验核心专利权利要求/同族/被引，再用 `patent_valuation_scorecard` 读取价值分和维度拆解。
6. 补公开市场信号：当 MCP 不能回答政策、招投标、技术交易、企业扩产、融资、招聘、产品线或采购需求时，检索公开 Web 来源。只把权威来源写入结论，普通媒体和社交爆料只作为弱信号。
7. 匹配买方和路径：结合 `corporate_intangible_assets.potential_buyer`、相似专利申请人、产业链企业、招投标主体、园区招商对象和公开企业信息，输出“潜在线索”，不得写成“已有意向”。
8. 分层决策：A 类立即外推，B 类补证培育，C 类组合运营，D 类防御维护，E 类放弃/归档候选。高估值但交易洁净度异常时优先降级。
9. 生成客户成果物：默认输出 HTML 报告；用户指定或本地可稳定渲染时同步输出 PDF、Word 或 Excel。面向客户的成果必须能独立打开，能直接给管理层、商务、研发、法务或 IP 团队使用。
10. 补专利详情链接：生成 HTML/Word/PDF 前，抽取正文、表格和招商卡中的公开/公告号，用 `global_core_patent_database.search_patents` 或 `advanced_patent_search.search_patents_nested` 获取 `patent_id`，将专利号链接到智慧芽 Eureka 详情页；用 `patsap_patent_search.patsnap_fetch` 抽样校验至少 1 条链接可被 MCP 识别。
11. 质检：逐条检查来源、证据等级、敏感信息、夸大表述、不可执行动作、专利号链接覆盖率、访问权限提示和免责声明。

## 子代理分工

资产超过 50 件、跨 3 个以上技术领域、需要同时查证与生成成果物，或用户明确要求多代理时，拆成子代理：

- 资产清洗代理：主体归一、资产去重、法律状态和权属问题标记。
- 技术证据代理：使用三层搜索 MCP 输出技术证据表和创新边界。
- 市场信号代理：检索政策、招投标、交易平台、投融资、官网和新闻，标注来源等级。
- 买方匹配代理：生成买方池、排除名单和触达优先级。
- 交易策略代理：结合价值评分与交易洁净度，给出许可、转让、联合开发、质押、维护或放弃建议。
- 视觉报告代理：按客户级视觉规范生成 HTML/PDF/Word/Excel。
- 质检代理：检查缺证、夸大、敏感泄露和不可执行动作。

若当前环境不能真正启动子代理，按这些角色顺序自检，并在报告中保留分工痕迹。

## 评分与降级

默认 100 分：技术差异化 18、商业需求强度 18、资产完整度 14、交易洁净度 14、买方匹配度 14、转化准备度 12、组合可运营性 6、国际化可用性 4。

分层：85-100 为 A 类，70-84 为 B 类，55-69 为 C 类，40-54 为 D 类，0-39 为 E 类。

硬性降级：

- 权属不清，最高 B 类。
- 无明确应用场景，最高 C 类。
- 少于两类证据，最高 C 类。
- 保密、出口管制、伦理或生物安全限制未核验，最高 D 类。
- 已到期或失效且无 know-how/商标/软著配套，最高 D 类。
- `patent_valuation_scorecard` 高分但 `patent_monetization_valuation` 显示法律/交易状态异常，按交易洁净度降级。

## 输出要求

按用户任务选择输出。默认交付一个客户可打开的 HTML 报告，并在最终回复中附路径；若用户明确要求或环境支持，再导出 PDF/Word/Excel。报告正文默认包含：

- 管理层摘要：资产规模、A/B/C/D/E 分层、90 天可推进线索、最值得外推的 3-5 个资产包、决策事项、主要风险。
- 资产盘活主表：排名、资产包、关联资产、应用场景、评分分层、推荐路径、潜在买方类型、证据强度、交易洁净度、下一步动作。
- 重点招商卡：一句话卖点、客户痛点、技术方案、证据链、适用行业、买方画像、合作模式、首轮外推话术、NDA 前后材料。
- 买方触达表：买方名称或类型、匹配理由、证据来源、优先级、禁触达事项、触达话术、预期下一步。
- 30/60/90 天行动计划：责任人、动作、所需材料、验收标准。

若成果物中出现专利公开号/公告号，默认将专利号做成可点击链接。优先链接至智慧芽 Eureka 详情页，格式为 `https://eureka.zhihuiya.com/view/#/fullText'figures/?patentId={patent_id}`；`patent_id` 来自 `global_core_patent_database` 或 `advanced_patent_search` 的检索结果。报告方法说明中必须提示：专利详情页通常需要已登录账号或机构权限。若无法获取 `patent_id`，保留纯文本专利号，并在质检说明中列出未链接清单。

面向客户的视觉成果使用“企业服务优先的瑞士国际主义极简科技风”：中性色、网格、强对齐、克制高亮、证据标签、评分条、时间轴和清晰表格。详细输出规范见 `references/output-spec.md`。

## 公开来源规则

优先级：政府/部委/国家知识产权局/官方公共平台 > 技术交易平台/招投标/上市公司公告/企业官网 > 行业协会/权威媒体/园区公告 > 普通媒体/论坛/社交平台。公开来源必须记录 URL、访问日期、来源等级和被使用结论。

## 示例问题

可直接向用户推荐 `references/example-prompts.md` 中的问题。最典型的起手式是：

```text
使用 $sleeping-patent-asset-activation，以「某公司」为对象，盘点其沉睡专利资产，识别最值得 90 天内盘活的资产包，并生成可独立打开的 HTML 报告；如果环境支持，同时导出 PDF。报告需包含资产分层、潜在买方、推荐路径和管理层摘要。
```

## 重要边界

不要把授权专利直接等同于高价值资产。不要把潜在买家写成确认意向。不要输出未经授权的客户名称、成本底线、未公开路线、配方或图纸。结论不构成法律意见、正式资产评估报告或交易承诺。

## 使用前配置
本 Skill 依赖智慧芽开放平台 MCP 服务：
- 完成安装、初次使用时需进行自检，参见 README.md
- 用户需完成账号授权，并确保 Agent 环境已启用对应 MCP 工具
- 若未完成配置，本 Skill 只能提供分析框架，无法检索实时数据或生成基于数据库的结论
- 缺少MCP配置时，引导用户参照 README.md 在 [[open.zhihuiya.com](https://open.zhihuiya.com/)](https://open.zhihuiya.com/) 获取MCP。
