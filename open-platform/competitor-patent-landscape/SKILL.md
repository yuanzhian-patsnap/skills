# 竞争对手专利布局策略调研技能

## 目标用户

企业 IP 团队、研发工程师、战略分析师。

## 输入参数

| 参数 | 是否必填 | 说明 |
|------|----------|------|
| 竞争对手名称 | ✅ | 如"华为"、"三星" |
| 技术领域 | ✅ | 如"折叠屏手机"、"固态电池" |
| PatSnap 检索式 | 可选 | 若不填则自动构造 |
| 目标市场 | 可选 | CN/US/EP/WO/JP，默认全球 |
| 产品示例名称 | 可选 | 用于产品图可视化，如"Mate X5" |
| Top N 核心专利数量 | 可选 | 默认 10 |

## 工作流程

### Step 1 — 网络搜索技术框架
- 搜索竞争对手在该技术的公开研究重点、产品特性、技术路线
- 归纳 3-6 个技术子方向，构建技术框架

### Step 2 — 专利检索
- 使用 PatSnap 检索式检索专利，获取最多 100 条结果（总量信息保留）
- 若无检索式则自动构造：`ANCS:(公司名) AND TA_ALL:(技术关键词)`

### Step 3 — 核心专利筛选（简单同族数量维度）
- 批量 fetch 候选专利的 family 模块，提取 `Patent Simple Family Members` 数量
- 同族数量越多 = 多国同步布局 = 优先级越高
- 筛选标准：简单同族 ≥ 3 件视为核心专利；1-2 件视为外围或区域性专利
- 输出 Top N 核心专利清单，含同族成员摘录

### Step 4 — 专利内容深度阅读
- 对 Top N 专利 fetch basic 模块，阅读权利要求和摘要
- 将每件专利映射到 Step 1 的技术子方向
- 判断核心（宽权项、多国授权）vs 外围（窄权项、补充保护）

### Step 5 — 技术热力分布分析
- 统计各技术子方向的专利数量与同族数分布
- 识别：壁垒区（高密度+核心专利）、关注区（中密度）、白地机会区（低密度）

### Step 6 — 产品图可视化
- 在报告中嵌入目标产品的 SVG 示意图（用户指定或自动选择典型产品）
- 将技术子方向 + 对应核心专利映射到产品部件位置（如铰链区、屏幕区、散热区等）
- 每个区域标注对应专利公开号和链接

### Step 7 — 市场布局分析
- 统计各市场（CN/US/EP/WO/JP）的专利数量和授权状态
- 识别主力市场、重点同步市场、外观专利布局市场

### Step 8 — 策略总结
从以下维度输出结构化结论：
- **技术维度**：核心技术壁垒、布局分层逻辑、前沿方向
- **市场维度**：主力市场策略、同步扩展策略、区域差异化
- **IP 维度**：核心+外围圈层设计、代际延续策略、多主体申请
- **研发建议**：白地方向、FTO 重点、规避设计提示

### Step 9 — 生成 HTML 报告
输出包含以下模块的完整 HTML 报告：
1. 报告头（标题、检索式、统计总览）
2. 核心专利清单表格（同族数量排序）
3. 技术热力分布（色彩热力卡片）
4. 产品图可视化（SVG + 专利图例）
5. 全球市场布局（市场卡片）
6. 布局策略总结（技术/市场/IP/建议四卡片）
7. 研发团队重点提示

> 如需 PDF：用浏览器打开 HTML → Ctrl+P → 另存为 PDF

## 输出文件

- `huawei_foldable_patent_landscape.html`（或根据公司+技术命名）：完整报告
- 报告路径：`@session/competitor-patent-landscape/output/`

## 质量标准

- 所有专利链接均使用 PatSnap 真实 URL
- 同族数量来源于 Patent Simple Family Members 字段（不估算）
- 产品图 SVG 使用标准矢量绘制，清晰标注区域编号与颜色图例
- 策略结论每条均附证据（专利号 + 来源说明）
- 未经核实的数量结论标注为 Unverified
