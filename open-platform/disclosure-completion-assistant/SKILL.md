# 交底书智能补全助手

## 作品定位

本 Skill 面向专利代理机构及其服务的研发客户，帮助发明人把零散、口语化、产品化的技术描述整理成代理人可继续使用的结构化交底材料。

典型输入包括：

- 一句初始想法；
- 产品说明书片段；
- 论文摘要；
- 研发聊天记录；
- 尚未整理成专利交底书的技术说明。

## 核心场景

专利代理人在新案接洽时，经常收到不完整的技术材料。客户可能只说“系统可以自动推荐”“算法更准”“设备能提前预警”，但缺少专利撰写需要的关键信息，例如：

- 现有技术怎么做；
- 现有方案有什么缺陷；
- 本发明的核心技术手段是什么；
- 系统包含哪些模块或步骤；
- 技术效果是否有数据支撑；
- 希望重点保护什么。

本 Skill 将资深代理人的交底追问经验固化为步骤化引导，减少代理人与客户之间的反复沟通。

`assets/disclosure-guide.html`

页面名称：专利交底书智能引导系统。

该页面适合代理机构发送给发明人或研发人员填写，也适合销售演示时直接展示。

**重要：当用户要求查看、打开、生成或交付网页成果物时，必须直接使用 `assets/disclosure-guide.html` 作为最终网页。不要根据本 Skill 文档重新设计页面、重新生成 HTML、替换布局、改写交互或改变视觉风格。该 HTML 是已经确认的成品页面。**

如果运行环境需要输出网页文件，应复制 `assets/disclosure-guide.html` 到目标位置；如果运行环境需要展示页面，应打开该文件；如果运行环境要求说明成果物，应描述该文件的功能，而不是重新创建一个新页面。

## 页面流程

页面采用 7 步引导流程：

1. 基本信息：填写发明名称、技术领域、发明人和申请人。
2. 技术背景：说明现有技术和已有方案缺陷。
3. 核心发明：提炼核心创新点与技术效果。
4. 技术方案：描述系统架构、关键步骤、参数和变体。
5. 实施方式：用具体场景说明方案如何运行。
6. 保护目标：说明希望重点保护的方法、系统、模型、设备或数据处理方式。
7. 生成导出：生成结构化交底书草稿。

页面还包含开始页和中英文切换能力。

## AI 补全能力

每个可填写字段都提供 `AI 补全` / `AI Complete` 按钮。

设计目标：

- 研发人员可以先简单写一句粗略内容；
- AI 将粗略内容补成更接近交底书表达习惯的文字；
- 补全结果不是最终定稿，而是供客户确认和修改；
- 代理人拿到的是更完整、更结构化的草稿，而不是零散聊天记录。

示例：

研发输入：

> 系统根据用户行为推荐方案。

AI 补全后：

> 本系统基于用户历史行为、当前需求和候选方案属性构建多维特征，并根据特征匹配结果生成推荐方案，从而提升推荐匹配度并减少人工筛选时间。

## 输出结构

最终生成的交底书草稿包含：

1. 发明名称
2. 技术领域
3. 背景技术
4. 现有技术缺陷
5. 核心发明点
6. 技术效果
7. 技术方案
8. 具体实施方式
9. 保护目标
10. 公开状态

未填写部分会标记为“待补充”，便于客户和代理人继续完善。

## 使用方式

在 Eureka Desktop 中导入本 `.skill` 文件后，打开 Skill 中的 HTML 成果物即可体验。

推荐演示路径：

1. 点击“填充示例”快速进入示例状态；
2. 切换中英文查看双语能力；
3. 在任一字段中输入简单描述；
4. 点击 `AI 补全` 查看补全效果；
5. 进入“生成导出”查看结构化交底书草稿。

如需导出参赛成果物，请优先导出或引用 `assets/disclosure-guide.html`。该文件是客户可直接打开使用的最终成果物。

## 注意事项

当前 HTML 中的 `AI 补全` 为前端演示逻辑，用于说明产品流程和交互效果。真实上线时，应接入后端大模型接口，由后端安全调用模型 API，前端不暴露 API Key。
