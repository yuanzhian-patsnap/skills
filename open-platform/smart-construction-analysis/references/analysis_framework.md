# 智能建造技术分析框架参考

## 六大分析维度

1. **技术体系解构** - 核心技术栈、TRL评估、子域关联
2. **专利布局分析** - 申请人分布、IPC分类、引用网络
3. **学术研究热度** - 高被引文献、机构合作、转化路径
4. **核心主体竞争格局** - 国内外主体对比、技术壁垒评估
5. **成果转化与工程案例** - 典型项目效率数据、展会亮点
6. **趋势研判与挑战** - 无人化路线图、AI渗透、标准化瓶颈

## 检索策略三路并行

```
路径A: 语义检索 (semantic)
  - semantic_query: 描述技术原理或解决问题
  - 适用: 前沿方向探索、机理性技术

路径B: 关键词检索 (keyword)
  - keywords: 3-8个核心术语
  - 适用: 特定技术名称、产品标准

路径C: 申请人+时间过滤 (filter)
  - assignees + date_from/date_to
  - 适用: 特定主体竞争情报
```

## 证据分级标准

| 等级 | 来源 | 适用场景 |
|------|------|--------|
| Tier A | 授权专利 + 高被引论文 | 核心技术事实断言 |
| Tier B | 公开专利 + 一般论文 | 技术方向判断 |
| Tier C | 商业报道 + 展会 | 市场/商业化状态 |
| Tier D | 行业资讯 | 趋势参考，需标注低置信度 |

## 报告质量控制checklist

- [ ] 所有核心结论有 [S#] 来源标注
- [ ] 专利引用格式: `[专利号](智慧芽链接) [S#]`
- [ ] matched_total / returned_count 直接引用工具输出，不手写
- [ ] 负面结论已完成全量分页验证
- [ ] 竞争力矩阵评分有证据支撑
- [ ] Word报告通过 python-docx 生成，不内嵌完整HTML
