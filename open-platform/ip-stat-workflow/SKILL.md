# 知识产权成果统计工作流程助手

## 技能定位

本技能面向**科技管理部门负责人、知产专员、研发管理人员**，用于执行知识产权成果的全流程统计工作。

触发场景：
- 用户说"帮我做知识产权统计"、"知产成果汇总"、"IP台账核验"
- 用户提供单位名称或专利公开号列表，需要采集/核验/评级
- 用户需要生成年度IP报告、季度简报或上报材料

---

## 输入信息

| 输入项 | 必填 | 说明 |
|--------|------|------|
| 单位名称（含曾用名） | ✅ | 与CNIPA登记名称一致，用于patent.search的assignees参数 |
| 统计周期 | ✅ | 如"2024年度"、"2025Q2"，确定date_from/date_to范围 |
| 专利公开号列表 | 条件必填 | 用于数据核验（patent.fetch），台账中提取 |
| 项目清单（编号、名称、起止时间、技术关键词） | 可选 | 用于项目关联分类 |
| 竞争对手名称列表 | 可选 | 用于竞对对标分析 |
| 当前执行阶段 | 可选 | 用户可指定从哪个阶段开始，如"只做质量评级" |

---

## 工作流程（7个阶段）

### 阶段1：数据采集
- 调用 `patent.search`（filter策略）采集本单位全量在申/有效专利
- 与内部台账比对，输出差异清单

### 阶段2：数据核验
- 调用 `patent.fetch`（basic+legal）批量核验法律状态
- 调用 `patent.fetch`（basic+citation）更新被引用次数

### 阶段3：分类整理
- 按类型（发明/实用新型/外观/软著）、项目关联、权属（独有/共有/受让）分类
- 调用 `patent.search`（keyword+filter）补充识别项目关联专利

### 阶段4：质量评级
- 套用S/A/B/C四级评级规则
- 调用 `patent.fetch`（basic+family）核验多国布局（S级判定）
- 调用 `patent.search`（semantic+filter）评估技术覆盖密度

### 阶段5：汇总统计
- 输出年度统计表（总量/质量/转化/项目关联四维指标）
- 可选：调用 `patent.search`（filter）执行竞对对标

### 阶段6：上报归档
- 按上报对象（主管部门/国知局/项目验收方/内部管理层）生成对应材料
- 按命名规范归档统计结果与检索记录

### 阶段7：动态维护
- 提供月度/季度/半年/年度巡检清单
- 响应触发事件（授权公告、年费预警、转化合同签订等）

## 智慧芽MCP工具调用规则

| 工具 | 策略 | 场景 | 关键参数 |
|------|------|------|----------|
| `patent.search` | filter | 采集本单位全量专利 | assignees, legal_status, jurisdiction |
| `patent.search` | filter | 竞对对标 | assignees(竞对), date_from/to |
| `patent.search` | keyword+filter | 项目关联补充识别 | keywords, assignees, date_from/to |
| `patent.search` | semantic+filter | 技术覆盖密度评估 | semantic_query, assignees |
| `patent.fetch` | — | 法律状态核验 | keys(公开号), module: basic+legal |
| `patent.fetch` | — | 被引用次数采集 | keys(公开号), module: basic+citation |
| `patent.fetch` | — | 多国布局核验 | keys(公开号), module: basic+family |

> patent.fetch 每批建议≤20条，超出分批调用。

## 质量评级规则

| 等级 | 判断条件 |
|------|----------|
| S级（战略级） | PCT已授权 / 已转化实施 / 涉诉主动保护 / 海外多国布局≥3国 |
| A级（高价值） | 发明专利已授权 + 被引用次数≥3次 |
| B级（有效资产） | 已授权 + 年费维持中 + 不满足A级条件 |
| C级（待观察） | 申请中 / 实用新型 / 外观设计 |

---

## 执行守则

1. **逐阶段确认**：每完成一个阶段，输出该阶段结果并询问是否继续下一阶段。
2. **批量调用控制**：patent.fetch 每批≤20条，自动分批处理。
3. **数据溯源**：所有统计数据须标注来源（工具调用结果 / 用户提供 / 手工录入）。
4. **差异处理**：遇到台账与数据库不一致时，列出差异项，提示用户确认后再更新。
5. **脱敏提醒**：生成对外报送材料时，提示用户核查商业敏感信息并脱敏。
