# 科技成果转移转化供需匹配技能 v4.0

## 定位与触发

**用途**：对一项科技成果进行全流程转移转化供需匹配分析，输出结构化报告（含 ECharts 可视化）和潜在接受方排名名单。

**触发关键词**：供需匹配、技术转移、成果转化、转移转化分析、技术对接、成果匹配

**不适用场景**：
- 单纯专利检索或无效分析（请用其他技能）
- 无技术转移诉求的市场调研
- 纯商业尽调（无技术内容）

---

## 输入模式

| 模式 | 触发条件 | 说明 |
|------|----------|------|
| **模式 A** | 用户上传文档/图片/粘贴技术描述 | 直接提取技术内容，跳过专利检索 |
| **模式 B** | 仅提供公司名称或机构名称 | 先用 `patent.search` 检索 Top-5 高引用专利，再进入步骤 1 |

> 判断规则：同时有文档+公司名时，优先走模式 A，公司名作为技术来源字段。

---

## 工作流 6 步骤

### 步骤一：技术内容分析

提取并结构化以下字段：

| 字段 | 说明 |
|------|------|
| 技术来源 | 供方机构/公司/发明人 |
| 技术主题 | 一句话概括 |
| 技术简介 | 200字以内，原理与方法 |
| 核心优势 | 3~5 条差异化优势 |
| 知识产权状态 | 专利号/申请号/保密/开放许可意愿 |
| 应用场景 | 3~5 个具体可落地方向 |
| 成熟度等级 | TRL 1~9（技术准备水平） |

**内容丰富度要求（v4.0 新增）**：

步骤一必须包含以下子模块，内容详尽，不得简化：

1. **基础信息表**：来源机构、技术名称、TRL、知识产权、核心发明人
2. **技术背景与问题痛点**：描述该技术所解决的核心痛点，说明传统方案的局限性（100字以上）
3. **核心技术创新点**：列出 4 条以上具体创新点（含技术机制描述）
4. **KPI 关键性能指标看板**：以卡片形式展示 4 个以上核心技术参数（数值 + 单位 + 说明）
5. **竞品对比表**：与 3 个以上竞品/替代方案全维度对比（至少 5 列：方案名、核心指标1、核心指标2、深度/规模验证、国产化）
6. **应用场景 + 转移优势双栏**：左侧列出 5 个以上应用场景，右侧列出 5 条以上技术转移优势

---

### 步骤二：技术先进性评估（满分 100 分）

八因子评分，输出分项得分、权重得分、评语、总分及等级（L1~L5）：

| # | 评估因子 | 权重 |
|---|----------|------|
| 1 | 技术原理突破性 | 15% |
| 2 | 核心方法新颖性 | 10% |
| 3 | 技术可替代性（反向）| 10% |
| 4 | 技术壁垒高度 | 10% |
| 5 | 工艺兼容性 | 15% |
| 6 | 产业化成熟度 | 15% |
| 7 | 技术稳定性 | 10% |
| 8 | 规模化潜力 | 15% |

等级：L1 待观察 / L2 有价值 / L3 较先进 / L4 高价值 / L5 突破性

**内容丰富度要求（v4.0 新增）**：每个因子必须附带 1~2 句评分理由说明，解释为什么给此分值。

---

### 步骤三：智慧芽专利价值评估

调用 `patent.search` / `patent.fetch`，按四维度评估：

#### 技术价值
- 权利要求覆盖广度
- 被引用次数（forward citations）
- 基础性/平台性技术判断

#### 市场价值
- 目标市场规模（TAM 估算）
- 同族专利国家覆盖数
- 商业化进展证据

#### 法律价值
- 专利有效性状态与剩余年限
- 权利要求稳定性
- 专利质量指数（PQI）

#### 战略价值
- 竞争壁垒构建能力
- SEP 潜力判断
- 许可/转让/入股可操作性

**专利价值基准**：

| 类型 | 估值区间 | 系数 |
|------|----------|------|
| 普通发明专利 | 50~200 万元 | 1.0× |
| 核心发明专利 | 200~500 万元 | 1.0× |
| 无重稀土/关键材料 | 基础值 × 1.8 | 稀缺溢价 |
| PCT/多国同族 | 基础值 × 1.3~2.0 | 按国家数 |
| SEP 候选 | 基础值 × 2.5~5.0 | 需标准关联验证 |

**步骤三内容模块（v4.0 新增，必须全部输出）**：

1. **四维评分卡**：技术/市场/法律/战略各给出百分制评分 + 进度条 + 一句评分说明
2. **估值高亮框**：以醒目样式展示估值区间（含估值依据说明：专利类型 × 数量 × 溢价系数）
3. **估值依据 + 市场机会双栏**：左侧说明估值计算依据，右侧说明市场规模与增长趋势
4. **专利组合深度分析**（核心模块，必须包含）：
   - **专利组合统计卡**：检索命中数量、核心发明数量、国家标准数量
   - **IPC 技术分类分布**：按 IPC 大类展示专利分布，每类含代码、说明、件数（至少 4 个 IPC 类）
   - **关键专利逐条分析列表**：每条专利包含：专利类型标签、专利标题（可点击智慧芽链接）、专利号+申请日+公开日、摘要解读（2~3句）、IPC+法律状态+关键词标签、申请年份+引用次数
   - **专利战略洞察**：4 条以上战略建议（PCT 布局 / 时序梳理 / 打包策略 / 空白机会）

### 步骤四：潜在接受方挖掘 — 三维度二层加权模型

> 调用脚本：`scripts/match_scorer.py`

#### 总体权重

```
接受方匹配度总分（100分）
├── 专利信息匹配    40分
├── 公开新闻信息    30分
└── 招投标信息匹配  30分
```

#### 匹配等级

| 总分 | 等级 | 建议 |
|------|------|------|
| 85~100 | ⭐⭐⭐⭐⭐ 强烈推荐 | 立即跟进，优先级最高 |
| 70~84 | ⭐⭐⭐⭐ 高度匹配 | 重点跟进，准备方案 |
| 55~69 | ⭐⭐⭐ 中度匹配 | 常规跟进，持续观察 |
| 40~54 | ⭐⭐ 低度匹配 | 列入观察名单 |
| <40 | ⭐ 弱匹配 | 暂不优先 |

**接受方卡片内容丰富度要求（v4.0 新增）**：

每家接受方卡片必须包含：
- 企业名称 + 匹配度总分（百分制）
- 匹配等级标签（强推荐/推荐跟进/待观察）
- 行业/规模标签
- 核心推荐理由（3 条以上具体说明，含数据支撑）
- 建议跟进动作（1~2 条可操作的具体行动）

---

### 步骤四·五：技术转移风险评估（v4.0 新增，必须执行）

**四大风险维度**（各占 25 分，满分 100 分）：

| 维度 | 核心评估点 |
|------|-----------|
| 🔬 技术风险 | 成熟度不足、依赖单一发明人、复现难度、技术迭代速度 |
| 📈 市场风险 | 需求不确定性、竞品替代威胁、渗透周期、政策变动 |
| ⚖️ 法律风险 | 专利有效性争议、权属纠纷、侵权风险、合同条款 |
| 🤝 合作方风险 | 技术消化能力、财务稳健性、合作意愿真实性、信息不对称 |

**综合风险等级**：

| 风险指数 | 等级 | 推荐策略 |
|---------|------|---------|
| 0~20 | 🟢 低风险 | 正常推进，无需特别管控 |
| 21~40 | 🟡 中低风险 | 重点关注法律条款，建议律所介入 |
| 41~60 | 🟠 中高风险 | 先小规模试点，设置里程碑节点 |
| 61~80 | 🔴 高风险 | 专业机构介入，分阶段推进 |
| 81~100 | ⛔ 极高风险 | 暂缓，优先解决核心风险项 |

**步骤四·五报告可视化要求**：
1. **四维雷达图**（ECharts）：展示技术/市场/法律/合作方四维风险值，同时展示"理想区间"对照系列
2. **雷达图图例布局**：图例放在雷达图右侧，上下排列，使用 HTML 自定义图例，不使用 ECharts 内置 legend，不得与轴标签重叠
3. **雷达图容器**：flex 布局，左侧图表 + 右侧图例面板并排；容器高度不低于 320px
4. **Top-3 接受方个性化风险画像卡片**：每张卡片含风险等级标签 + 3 条个性化关注点（内容各不相同，不得重复）

### 步骤五：报告生成（HTML 可视化）

> 调用脚本：`scripts/report_generator.py` → `generate_html_report()`

**报告必须包含以下模块**：

|------|-----------|

| 技术内容摘要 | 结构化卡片（含背景、创新点、KPI看板、竞品对比、应用场景双栏）|
| 先进性评估 | 进度条 + 分项评分卡（2列网格）+ 大号总分展示 |
| 专利价值 | 四象限卡片 + 估值高亮框 + 专利逐条分析列表 |
| 接受方排名 | 卡片列表（含匹配分、标签、推荐理由、建议动作）|

**⚠️ 桌面保存强制规则**：

- 同时在 `@session/reports/` 保留备份
- 文件名格式：`{主题简称}_成果转化报告.html`

## 执行节奏

1. 确认输入模式（A / B）
2. 步骤一→四顺序执行，输出中间结果
3. 步骤四·五紧接步骤四执行，不得省略
4. 步骤五~六用户确认后统一执行
5. 报告生成后自动保存到桌面

## 质量控制

- 所有匹配得分须有可追溯数据支撑，不得捏造企业信息
- 专利估值须注明基准类型和溢价系数
