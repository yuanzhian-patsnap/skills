# 通用行业 FTO 报告要求

## 风险点文档结构

输入 Word 文档通常包含：
- `风险点整体描述` 或 `技术方案`：标的产品/技术方案的自然语言说明。
- 可选项目信息：产品名称、项目名称、行业、分类号、竞争对手、区域、法律状态。
- `风险点技术特征描述` 表格：列为 `技术特征类型`、`技术特征描述`、`技术特征关键词`。

将包含 `重要`、`核心`、`必要`、`关键` 的行视为重点技术特征。其他行作为普通技术特征参与 feature_pair 检索和后续权利要求比对。

## 可识别输入字段

字段可写在段落中，例如 `竞争对手：A公司、B公司`，也可写在两列表格中。

推荐字段：
- 产品名称 / 标的产品
- 项目名称
- 行业
- 分类号 / IPC
- 主分类号 / MIPC
- 竞争对手 / 竞争对手名称 / 排查对象 / 检索对象
- 区域 / 国家地区 / AUTHORITY
- 法律状态

命令行参数优先级最高，其次是 Word 文档字段，最后是 `references/config.json`。

## 检索与比对边界

通用行业不得沿用服务器滑轨的固定检索限制。仅在用户、Word 文档或配置提供相应字段时拼接：
- `IPC:(...)`
- `MIPC:(...)`
- `ALL_AN:(TREE@"...")`
- `SIMPLE_LEGAL_STATUS:(...)`
- `AUTHORITY:(...)`

用户必须直接提供检索式。通过 `--query` 或 `--queries-json` 输入的检索式应直接运行，不要自动改写、追加过滤条件或根据 Word 文档自动构建。仍需将检索式写入 `queries.json` 并在报告中披露。

每件进入侵权比对并获得风险结论的专利必须有 P018 返回的权利要求1全文。没有 claim 1 的专利只进入阻断/待核查清单。

## AI07 比对输出字段

要求 AI07 针对每件专利输出：
- 专利公开号、标题、权利要求1来源。
- 标的产品技术特征是否落入权利要求1。
- 字面比对结论。
- 等同比对结论。
- 差异技术特征。
- 风险等级：High / Medium / Low / 待核查。
- 报告用结论句。

AI07 仅作为辅助记录。报告结论以 P018 权利要求1和结构化特征比对为准。

## FTO 报告结构

参考 `assets/FTO报告模板.docx`，报告应至少包括：
- 封面与报告日期。
- 高相关专利清单。
- 总体结论。
- 分析目的。
- 分析范围：区域、竞争对手、分类号、法律状态、检索式模式。
- 产品/技术方案介绍。
- 检索策略与检索式。
- 专利筛选与标引结果。
- 风险结论分析。
- 重点专利逐件比对。
- 阻断/待核查专利附录。
- 免责声明。

结论优先，措辞保持法律风险分析报告风格。不要把缺少 claim 1、缺少法律状态信息或缺少 AI07 输出的专利写成“无风险”。
